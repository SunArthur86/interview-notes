# 你对当下Agent能力怎么看待？

## Agent能力成熟度评估

```
能力维度          水平        评估
─────────────────────────────────────
任务拆解          ★★★★☆      显著进步
Tool调用          ★★★★☆      可靠性提升
知识检索(RAG)     ★★★★☆      成熟
长程推理          ★★☆☆☆      仍然不足
多步决策          ★★☆☆☆      稳定性差
自我纠错          ★★☆☆☆      能力有限
通用迁移          ★☆☆☆☆      基本做不到
```

## 已能稳定落地的场景

| 场景 | 代表产品 | 为什么能落地 |
|------|---------|------------|
| **客服Bot** | 各大平台 | 流程固定、容错高、有人工兜底 |
| **Research助手** | Perplexity/ChatGPT | 单轮检索+总结、不需要长链路 |
| **代码辅助** | Copilot/Cursor | 单文件级、有IDE反馈循环 |
| **数据分析** | NL2SQL+Chart | 输出结构化、可验证 |
| **文档处理** | 归档/分类/抽取 | 任务边界清晰、可批量校验 |

## 仍然困难的场景

```
困难场景的共同特征：

1. 长链路任务 → 错误率指数累积
   "帮我调研竞品并写一份50页报告"
   → 10步串行，每步90%→整体35%成功率

2. 需要精确推理 → 概率模型不擅长
   "分析这段代码的并发安全问题"
   → 需要确定性逻辑推理，LLM容易遗漏

3. 开放式探索 → 没有明确终止条件
   "帮我找到一个好的创业方向"
   → 评价标准模糊，无法自动判断完成

4. 跨系统协作 → 接口和状态同步复杂
   "同时操作CRM、ERP和邮件系统"
   → 每个系统API不同、状态不一致
```

## 核心瓶颈

```
┌─────────────────────────────────────────┐
│           Agent 三大瓶颈                │
├──────────────┬───────────┬──────────────┤
│   稳定性差    │  成本高    │  长链路易失败 │
├──────────────┼───────────┼──────────────┤
│ 同一输入不同  │ 每步调用   │ 10步串行      │
│ 输出不一致    │ LLM+Tool  │ 每步95%       │
│              │ 成本累积   │ 整体=60%      │
│ 无法保证SLA   │ 复杂任务   │ 中间任何一步   │
│              │ $1-5/次   │ 失败全盘失败  │
└──────────────┴───────────┴──────────────┘
```

## 面试回答要点

> "我觉得现在Agent能力进步很快，尤其任务拆解和Tool调用。

> **简单任务已经能稳定落地**，比如客服、Research、代码辅助。这些场景的共同特点是：流程短、容错高、有人工兜底。

> **但复杂任务问题还很多**，主要是稳定性差、成本高、长链路容易失败。一个10步的串行任务，即使每步95%成功率，整体也只有60%。

> 离真正通用Agent还有距离。现阶段更务实的做法是把Agent定位为特定场景的效率工具，而不是试图做万能助手。"

## 面试加分点

1. **辩证看待**：既肯定进步又指出局限，体现成熟的技术判断
2. **量化分析**：用错误率累积模型解释为什么长链路难
3. **场景化**：区分"能落地的"和"不能落地的"，并解释原因
4. **务实路线**：推荐"特定场景优先落地"而非"一步到位做通用Agent"

## 记忆要点

- 成熟度盘点：任务拆解与Tool调用成熟，长程推理与自我纠错依然薄弱
- 易落地场景：流程短、容错高、有人工兜底(如客服/代码辅助/数据简答)
- 长链路瓶颈：因为错误率指数级累积(10步x90%=35%)，所以多步决策极易全盘崩溃
- 务实落地观：现阶段做特定场景的效率工具，而非一步到位造通用Agent

# 训练一个14B参数的FP16模型，理论上需要多少显存？

> 来源：字节跳动大模型技术面试二面

## 理论显存计算

### 1. 模型权重

```
14B 参数 × 2 bytes (FP16) = 28 GB
```

### 2. 优化器状态 (AdamW)

AdamW 为每个参数维护4个副本：

```
┌──────────────────────────────────────────────────┐
│        每个FP16参数的显存开销 (AdamW)             │
├──────────────────┬────────┬──────────────────────┤
│     组成部分      │ 精度   │    bytes/参数        │
├──────────────────┼────────┼──────────────────────┤
│ 模型权重 W        │ FP16   │  2                   │
│ 梯度 ∇W           │ FP16   │  2                   │
│ AdamW m (一阶矩)  │ FP32   │  4                   │
│ AdamW v (二阶矩)  │ FP32   │  4                   │
│ FP32主权重 (混合) │ FP32   │  4                   │
├──────────────────┼────────┼──────────────────────┤
│ 合计              │        │  16 bytes/参数       │
└──────────────────┴────────┴──────────────────────┘

14B × 16 bytes = 224 GB  ← 仅权重+优化器!
```

> **注**：纯FP16训练（无FP32主权重）为12 bytes/参数 = 168GB。  
> 混合精度训练（推荐）为16 bytes/参数 = 224GB。

### 3. 激活值

激活值显存取决于前向传播中需要保存的中间结果：

```
激活值 ≈ batch_size × seq_len × hidden_dim × num_layers × bytes

以14B模型典型配置 (hidden=5120, layers=40):
  batch=1, seq=4096:
  激活值 ≈ 1 × 4096 × 5120 × 40 × 2 (FP16)
         ≈ 1.68 GB/层 × 40 ≈ 不对，实际更复杂
```

更精确的估算（Transformer每层激活值）：

```
每层激活 ≈ batch × seq × hidden × (34 × hidden + 5 × seq) × 2 bytes

使用梯度检查点后:
  激活值 ≈ batch × seq × hidden × √layers × 2 bytes
  (只保存√N层的激活，其余重算)
```

### 4. 总计

```
┌──────────────────┬─────────────┬────────────────────┐
│      组成部分     │   显存(GB)  │      占比          │
├──────────────────┼─────────────┼────────────────────┤
│ 模型权重 (FP16)   │    28       │    15%             │
│ 梯度 (FP16)       │    28       │    15%             │
│ 优化器状态 (FP32) │   168       │    60% ← 最大!     │
│ 激活值 (FP16)     │   ~60       │     ~10%          │
├──────────────────┼─────────────┼────────────────────┤
│ 总计              │  ~284 GB   │    100%            │
└──────────────────┴─────────────┴────────────────────┘

→ 约需 4× A100-80GB (320GB) 或 6× A100-40GB
```

## INT8 量化的节省

```
┌──────────────────┬───────────┬───────────┬──────────┐
│      组成部分     │  FP16 (GB)│ INT8 (GB) │  节省    │
├──────────────────┼───────────┼───────────┼──────────┤
│ 模型权重          │    28     │    14     │  14 GB   │
│ 梯度             │    28     │    14     │  14 GB   │
│ 优化器状态        │   168     │   ~168    │  ~0      │
│ 激活值           │    60     │    30     │  30 GB   │
├──────────────────┼───────────┼───────────┼──────────┤
│ 总计             │   284     │   ~226    │  ~58 GB  │
└──────────────────┴───────────┴──────────┴──────────┘

注意: 优化器状态仍需FP32精度，INT8主要节省权重和激活值
总节省约 20-25%
```

## 各部分显存占比可视化

```
训练显存分布 (14B模型, FP16, AdamW):

优化器状态  ████████████████████████████  60%
激活值     ████                           10%
模型权重   ███                            15%
梯度       ███                            15%
            ↑
     优化器状态是显存大头!
     → 这就是ZeRO优化的目标
```

## 显存优化手段

| 方法 | 节省部分 | 节省比例 | 代价 |
|------|---------|---------|------|
| **ZeRO-1** | 优化器状态分片 | ~4× | 通信增加 |
| **ZeRO-2** | +梯度分片 | ~8× | 通信增加 |
| **ZeRO-3** | +权重分片 | ~N× | 通信大增 |
| **梯度检查点** | 激活值 | ~√N | 计算增加33% |
| **混合精度** | 权重+梯度 | ~50% | 精度风险 |
| **LoRA/QLoRA** | 只训练少量参数 | ~99% | 效果略降 |

```python
# 实际配置示例: 14B模型在4×A100-80GB上训练
from transformers import TrainingArguments
from deepspeed import zero_to_fp32

training_args = TrainingArguments(
    output_dir="./output",
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    fp16=True,                    # 混合精度
    gradient_checkpointing=True,   # 梯度检查点
    deepspeed="ds_config_zero2.json",  # ZeRO-2
)

# ds_config_zero2.json
"""
{
  "zero_optimization": {
    "stage": 2,
    "offload_optimizer": {"device": "cpu"},  # 优化器卸载到CPU
    "allgather_partitions": true,
    "reduce_scatter": true
  },
  "fp16": {"enabled": true}
}
"""
```

**面试加分点**：提到Megatron-LM的TP+PP组合可以将14B模型放在8×V100-32GB上训练；提到Flash Attention减少激活值存储（IO-aware的精确注意力）；提到CPU offloading将优化器状态放在CPU内存中，用PCIe带宽换取GPU显存。

## 记忆要点

- 参数显存乘以16：AdamW混合精度训练中，每参数需16Bytes(权重2+梯度2+优化器状态12)。
- 优化器状态是大头：FP32的主权重、动量m和v占据12Bytes，占整体显存超一半。
- 总显存需四张卡：14B模型纯训练显存高达约224GB，需4张A100-80G才能装下。

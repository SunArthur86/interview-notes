# 美团 Agent 二面：知识库检索会怎么问到底？

## 一、面试官的真实考察点

```
面试官问"知识库检索怎么做"时，真正想考察的：

初级回答（不够）：
  "用向量数据库，把文档向量化，查询时算相似度"
  → 评委：只知道概念，没有深度

中级回答（及格）：
  "文档分块→向量化→存向量库→查询时检索top-k→Rerank→生成"
  → 评委：知道流程，但缺细节

高级回答（加分）：
  全链路 + 每环节的优化 + 实战经验 + 权衡分析
  → 评委：有体系有深度有实践
```

## 三、面试官常追问的问题

### 追问 1：分块怎么切？

```
"你chunk_size设多少？为什么？"

回答策略：
  "我们对比了200/500/1000字符，最终选500。
   太小（200）：语义不完整，检索到但信息不全
   太大（1000）：一个chunk混多个主题，检索不精准
   500是recall和precision的平衡点。
   
   进阶：我们用父子分块——检索用小块(200)，返回用大块(1000)，
   兼顾精准和上下文完整。"
```

### 追问 2：检索召回率低怎么办？

```
"如果相关文档检索不到，怎么优化？"

回答策略：
  "分三步排查：
   1. 查查询侧：是不是口语化导致语义鸿沟？→ 查询改写/HyDE
   2. 查检索侧：单一向量检索有盲区？→ 加BM25混合检索
   3. 查数据侧：分块有没有切断关键信息？→ 父子分块/上下文扩展
   
   我们线上从recall 60%优化到85%，主要是加了混合检索和Rerank。"
```

### 追问 3：怎么防幻觉？

```
"LLM编造了文档里没有的内容怎么办？"

回答策略：
  "三层防护：
   1. Prompt约束：明确要求'只基于文档回答，不知道就说不知道'
   2. 后处理校验：用faithfulness指标检测，答案的每个claim
      检查是否有文档支持，不支持的标注或删除
   3. 引用标注：要求LLM在每个关键信息后标注来源，
      无法标注的说明可能是编造
   
   线上我们faithfulness从70%提升到92%。"
```

### 追问 4：大规模知识库怎么办？

```
"百万级文档，检索性能怎么保证？"

回答策略：
  "分层处理：
   1. 索引优化：HNSW索引，ef_search调优，O(logn)查询
   2. 分区检索：按业务域/时间分区，先路由再检索
   3. 缓存：热点查询缓存结果，语义缓存（相似问题）
   4. 分布式：Milvus集群，分片+副本
   
   百万文档，P99延迟<50ms。"
```

### 追问 5：怎么评估效果？

```
"你怎么知道你的RAG效果好？"

回答策略：
  "用RAGAS框架，四个指标：
   - faithfulness：防幻觉（最重要）
   - context_recall：检索全不全
   - context_precision：检索准不准  
   - answer_relevancy：答案切不切题
   
   建立100个测试case的评估集，每次优化后回归测试。
   线上持续收集Bad Case，形成优化闭环。"
```

## 四、加分项：展现工程思维

```
面试中体现"工程化"的表达：

1. "我们做了A/B测试对比..."
   → 有数据驱动的意识

2. "考虑到成本，我们用模型分层..."
   → 有成本意识

3. "线上监控发现...，我们优化了..."
   → 有线上实战经验

4. "这个方案的tradeoff是..."
   → 能分析优劣势

5. "如果重新设计，我会..."
   → 有反思和改进意识
```

## 五、面试回答模板

```
"我们的知识库检索系统是这样的：

【架构】
数据层用父子分块+BGE Embedding，检索层用向量+BM25混合检索，
后接Cross-Encoder Rerank，生成层用GPT-4+引用标注。

【优化】
上线初期recall只有60%，通过三方面优化到85%：
1. 加了BM25混合检索（+15%）
2. 加了Rerank（+10%）  
3. 查询改写解决语义鸿沟（+5%）

【评估】
用RAGAS评估，faithfulness 92%，建立了Bad Case闭环。

【挑战】
最大的挑战是延迟——多路召回+Rerank会增加300ms，
我们通过并行化和缓存优化到P99<200ms。"
```

## 六、面试加分点

1. **别只答"向量检索"**：这是最低级回答，面试官会步步追问直到你答不出
2. **全链路 + 数据**：讲清每环节 + 给出具体数字（recall从X到Y），显得真实
3. **展现权衡思维**：不只说"用了什么"，还说"为什么这么选/有什么 tradeoff"

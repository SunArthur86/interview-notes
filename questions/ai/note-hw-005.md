# 【华为面经】优化器 Adam 与 SGD 的本质区别？大模型为什么用 AdamW？

## 一、优化器的演进脉络

```
SGD → Momentum → NAG → AdaGrad → RMSProp → Adam → AdamW
 ↓      ↓         ↓      ↓          ↓         ↓       ↓
朴素   加惯性   改惯性  自适应    改自适应  集大成   修bug
下山   方向     预判    学习率    学习率    动量+自适应  权重衰减
```

理解这条演进线，就能理解"为什么大模型用AdamW"。

## 二、SGD：最朴素的梯度下降

### 2.1 公式

```
θ_{t+1} = θ_t - η · ∇L(θ_t)
```

每个参数按"梯度反方向 × 固定学习率"更新。

### 2.2 SGD的问题

```python
# SGD在椭圆形loss等高线上的行为
# 想象一个狭长的山谷（一个方向陡，一个方向平缓）

import numpy as np
import matplotlib.pyplot as plt

# 椭圆loss: L(x,y) = 10x² + y²
def loss(x, y): return 10*x**2 + y**2
def grad(x, y): return np.array([20*x, 2*y])

# SGD轨迹
theta = np.array([5.0, 5.0])
lr = 0.08
trajectory = [theta.copy()]
for _ in range(50):
    g = grad(*theta)
    theta = theta - lr * g
    trajectory.append(theta.copy())

# 结果：在x方向(陡)剧烈震荡，y方向(平缓)前进缓慢
# 像走Z字形下山
```

**问题**：
1. **震荡严重**：陡峭方向梯度大，容易overshoot
2. **学习率难调**：全局一个lr，对陡峭方向太大，对平缓方向太小
3. **易陷局部最优**：没有"惯性"冲出浅谷

## 三、动量（Momentum）：引入惯性

### 3.1 公式

```
v_{t+1} = β · v_t + (1-β) · ∇L(θ_t)    # 一阶动量：历史梯度滑动平均
θ_{t+1} = θ_t - η · v_{t+1}              # 用动量更新
```

β（通常0.9）控制"惯性大小"：β越大越平滑，越小越接近纯SGD。

### 3.2 动量的作用

```
没有动量：每步只看当前梯度 → 震荡
有动量  ：每步方向 = 0.9×上一步方向 + 0.1×当前梯度
        → 在一致方向上加速，在震荡方向上抵消
```

像滚下山的球——有了质量就有了惯性，能冲过小坑、平滑震荡。

## 四、Adam：自适应矩估计

### 4.1 核心思想

Adam = **Adaptive** + **Moment** = 自适应学习率 + 动量

它同时维护两个量：
- **一阶动量 m**（mean）：梯度的滑动平均 → 方向
- **二阶动量 v**（uncentered variance）：梯度平方的滑动平均 → 该参数历史梯度大小

```
# Adam更新公式
m_t = β1·m_{t-1} + (1-β1)·g_t          # 一阶动量（方向）
v_t = β2·v_{t-1} + (1-β2)·g_t²         # 二阶动量（自适应步长）

m̂_t = m_t / (1 - β1^t)                 # 偏差修正（初期m/v偏小）
v̂_t = v_t / (1 - β2^t)

θ_t = θ_{t-1} - η · m̂_t / (√v̂_t + ε)   # 自适应更新
```

### 4.2 Adam的"自适应"如何工作

关键在 `m̂_t / √v̂_t` 这一项：

```
某参数历史梯度大 → v̂_t大 → √v̂_t大 → 步长缩小
某参数历史梯度小 → v̂_t小 → √v̂_t小 → 步长放大

效果：每个参数有自己的有效学习率，自动适配
- 频繁更新的参数（如embedding中热门词）：步长自动缩小
- 稀疏更新的参数（如冷门词）：步长保持较大
```

```python
# 直观对比：在稀疏embedding场景
# 词"the"出现10000次 → 梯度大 → Adam自动减小步长 → 避免过拟合
# 词"pterodactyl"出现2次 → 梯度小 → Adam保持步长 → 充分学习
# SGD用同一学习率，无法区分
```

### 4.3 偏差修正为什么重要

```
初始化：m_0 = 0, v_0 = 0

t=1时：
m_1 = 0.1 · g_1            # 偏小（应为g_1）
v_1 = 0.001 · g_1²         # 偏小很多（应为g_1²）

如果不修正：步长 = m_1/√v_1 = 0.1·g_1 / √(0.001·g_1²) ≈ 3.16
                  但初期我们期望步长≈g_1（接近1）

修正后：
m̂_1 = m_1 / (1-0.9) = g_1       ✓
v̂_1 = v_1 / (1-0.999) = g_1²    ✓
步长 = g_1 / √g_1² = 1           ✓ 正确
```

## 五、AdamW：修复权重衰减的bug

### 5.1 问题：Adam的权重衰减耦合bug

**权重衰减（L2正则）** 的本意：在loss里加 `λ·||θ||²`，让参数不要太大，防止过拟合。等价于在梯度上加 `λ·θ`：

```
# 理想的权重衰减
g' = g + λ·θ          # 梯度上加正则项
θ ← θ - η · g'
```

但在Adam里，这个 `λ·θ` 会被塞进二阶动量v：

```
# Adam的"权重衰减"（其实是L2正则）
v_t = β2·v_{t-1} + (1-β2)·(g_t + λ·θ)²    # λ·θ也参与平方平均
                                                          ↑
                                            问题：正则项被自适应学习率缩放
```

**后果**：不同参数的权重衰减强度不一致——梯度大的参数，正则被削弱；梯度小的参数，正则被放大。这违背了L2正则"统一约束所有参数"的初衷。

### 5.2 AdamW的解耦修复

```
# AdamW：权重衰减与梯度更新解耦
m_t = β1·m_{t-1} + (1-β1)·g_t     # 注意：g_t是纯梯度，不含λ·θ
v_t = β2·v_{t-1} + (1-β2)·g_t²
m̂_t, v̂_t = 偏差修正

θ_t = θ_{t-1} - η · m̂_t / (√v̂_t + ε)   # Adam的更新
        - η · λ · θ_{t-1}                 # 独立的权重衰减项
```

权重衰减直接作用于参数本身，与自适应学习率解耦——**每个参数的衰减强度一致**。

### 5.3 效果对比

```
实验：在GPT-2规模模型上的对比

Adam + L2正则（耦合版）：loss收敛，但泛化较差
AdamW（解耦版）        ：相同收敛速度，泛化更好

论文(Decoupled Weight Decay Regularization, Loshchilov & Hutter, 2017)
实证：ImageNet上Adam+L2比SGD差，但AdamW追平甚至超过SGD
```

## 六、大模型为什么用AdamW而非SGD

| 优化器 | 收敛速度 | 调参难度 | 泛化 | 显存 | 适用场景 |
|--------|---------|---------|------|------|---------|
| **SGD** | 慢 | 难（需调lr+momentum） | 好（flat minima） | 少 | CV、传统深度学习 |
| **Adam** | 快 | 易（默认参数即可） | 中 | 多（2倍动量状态） | NLP早期、RNN |
| **AdamW** | 快 | 易 | 好（正则正确） | 多 | **大模型标配** |

### 6.1 大模型用AdamW的原因

1. **参数规模巨大（数十亿到万亿）**：SGD的固定学习率无法适配这么多参数的不同特性，AdamW的自适应学习率是必需的
2. **训练成本极高**：AdamW收敛快，节省GPU/NPU时间。SGD慢一倍意味着训练成本翻倍
3. **稀疏参数多**：大模型的embedding层极度稀疏（词表几万到百万），AdamW的自适应步长对稀疏更新更友好
4. **正则化必须正确**：大模型容易过拟合，AdamW的正确权重衰减比Adam的耦合L2更有效

### 6.2 大模型训练中AdamW的典型超参

```python
# GPT-3 / LLaMA 等大模型的典型配置
optimizer = AdamW(
    lr=3e-4,           # 峰值学习率（配合warmup）
    betas=(0.9, 0.95), # 注意β2用0.95而非0.999（大模型惯例，响应更快）
    weight_decay=0.1,  # 权重衰减（解耦，比Adam的L2可以设大）
    eps=1e-8,
)

# 学习率调度：warmup + cosine decay
scheduler = CosineAnnealingWithWarmup(
    warmup_steps=2000,     # 前2000步线性warmup
    max_lr=3e-4,
    min_lr=3e-5,
    total_steps=100000,
)
```

## 七、为什么大模型必须warmup

Adam的初期方差估计 v̂_t 不稳定（样本少），如果一开始就用大学习率，会导致参数剧烈抖动甚至发散。Warmup的作用：

```
训练初期（前2000步）：
  - v̂_t估计不准（梯度方差大）
  - 学习率从小线性增长，给v̂_t积累样本的时间
  - 避免初期梯度噪声导致的参数崩坏

训练中期：
  - v̂_t估计稳定
  - 学习率达到峰值，快速收敛

训练后期：
  - cosine decay，学习率缓慢下降
  - 精细调整，找到更优解
```

```python
import torch

# 1. 传统CV任务（追求泛化极限）
optimizer_cv = torch.optim.SGD(

# 3. 大模型标配

    model.parameters(),

    betas=(0.9, 0.95),

```

## 加分点

1. **知道偏差修正的原因**：m_0/v_0初始化为0导致初期估计偏小，修正项 `1-β^t` 补偿
2. **能解释sharp vs flat minima**：SGD因噪声大能逃出sharp minima找到flat minima（泛化好），Adam收敛快但易陷sharp（这是Adam泛化不如SGD的理论解释）
3. **了解新优化器**：Lion（符号函数，省显存）、Adafactor（省二阶动量显存）、Sophia（用Hessian对角线）

## 雷区

- **混淆Adam的weight_decay和AdamW的weight_decay**：前者是耦合L2（有bug），后者是解耦正则（正确）——PyTorch的Adam也有weight_decay参数，但实现的是耦合版
- **忽视β2的选择**：传统任务用0.999，但大模型普遍用0.95——这影响二阶动量对近期梯度的敏感度
- **以为Adam不需要调学习率**：Adam对学习率相对鲁棒，但3e-4和1e-4效果差异巨大，仍需调

## 扩展

- **Lion优化器**（Google 2023）：用符号函数 `sign(m_t)` 替代 `m_t/√v_t`，省去二阶动量v，显存减半，效果相当
- **Sophia**（Stanford 2023）：用Hessian对角线估计替代二阶动量，理论上更接近二阶方法，收敛更快
- **学习率与batch size的关系**：大batch训练需线性放大学习率（sqrt scaling rule for Adam）

## 记忆要点

- 演进脉络：SGD 加惯性变 Momentum，再加自适应学习率演进至 Adam（动量+自适应）。
- Adam机制：维护一阶动量定方向，二阶动量做自适应步长，并用偏差修正解决冷启动偏小问题。
- 步长特性：历史梯度大则缩小步长防过拟合，梯度小则放大步长，等效给每个参数专属学习率。
- 为何用 AdamW：修复了 Adam 将权重衰减与梯度耦合的 Bug，实现真正的解耦权重衰减。

### 第一层：目标与动机

**Q：Adam 同时维护一阶动量 m 和二阶动量 v，每个参数都要存这两个状态，显存翻倍。大模型参数百亿到千亿，这个额外显存开销不是巨大浪费吗？为什么不用 SGD（零额外状态）省显存？**

因为 SGD 的固定学习率无法适配大模型参数的异质性。大模型里不同层的参数梯度特性差异巨大：embedding 层极度稀疏（冷门词偶尔更新一次，梯度小），attention 的 Q/K/V 矩阵稠密且梯度大。SGD 用一个学习率，对 embedding 太大（梯度小但 lr 不变，要么更新过猛要么几乎不动），对 attention 可能太小。Adam 的二阶动量 v 给每个参数"自适应步长"——梯度大的缩小步长、梯度小的放大步长，本质是"每个参数有专属学习率"。这个能力的代价是 2 倍参数量的额外显存（存 m 和 v），但换来的收敛速度提升（Adam 通常比 SGD 快 2-3 倍到达相同 loss）在大模型训练成本（数百万 GPU 小时）下远超显存代价。所以不是浪费，是"用显存换收敛速度和稳定性"的高性价比权衡。显存不够用 ZeRO/FSDP 分片优化，而非退回 SGD。

### 第二层：证据与定位

**Q：你说 Adam 的二阶动量 v 实现"自适应步长"。但如果某个参数在训练前期梯度一直很小（v 小，步长大），中期突然梯度变大，这时 v 还没更新上来，步长仍然很大，会导致参数震荡甚至发散。你怎么定位和解决这种"自适应滞后"问题？**

这是 Adam 的已知特性——二阶动量是滑动平均，对梯度的突变有滞后。定位方法是看训练 loss 曲线和梯度统计：如果 loss 在某个 step 突然飙升后无法恢复，且对应 step 有某些参数的梯度突增（看梯度范数监控），就是自适应滞后导致的发散。解法有几层：一是 warmup（前 2000 步小学习率），让 v 在学习率加大前积累足够样本，这是大模型标配；二是调大 β2（从 0.9 到 0.999），让 v 更平滑（但响应更慢，治标）；三是梯度裁剪（clip gradient norm，如 max_norm=1.0），直接限制梯度突增的破坏，即使 v 滞后也不会因为单次大梯度发散——这是大模型训练的必备保险；四是学习率调度配合（cosine decay 让中后期学习率下降，降低突变的影响）。实践中 warmup + gradient clipping + cosine decay 三件套是大模型稳定训练的标配，专门应对自适应滞后和梯度突变。

### 第三层：根因深挖

**Q：AdamW 你说修复了"权重衰减与自适应学习率耦合"的 bug。但这个"bug"从 Adam 提出就存在，为什么十几年才被发现和修复（AdamW 2017）？之前用 Adam 训的模型不也都 work 吗？**

因为"work"和"最优"是两回事。Adam 的耦合 L2 正则确实"work"——模型能收敛、效果不错，所以大家没紧迫感去质疑。但在大规模、强正则需求的场景（如 ImageNet 大训练、大语言模型）下，耦合的问题才暴露：正则强度随参数梯度大小变化，导致某些参数过拟合、某些过衰减，泛化不如 SGD。AdamW 的发现（Loshchilov & Hutter）是在"追求极致泛化"的实验中发现"Adam + L2 总是比 SGD + weight decay 差一点"，深挖才定位到耦合 bug。之前没发现是因为：一是小模型/短训练下差异不显著（模型容量小，正则差异被掩盖）；二是社区习惯"Adam 收敛快但不泛化好"的刻板印象，没深究原因；三是深度学习的经验主义（"work 就行"）压制了对理论一致性的追究。这反映了深度学习"先 work 再优化"的实用主义文化——很多 bug 在小规模下隐藏，scale up 后才暴露。

**Q：那如果 AdamW 的解耦权重衰减更好，为什么 β2 在大模型里常用 0.95 而不是 Adam 论文默认的 0.999？0.999 不是"更平滑更稳定"吗？**

因为 0.999 对大模型的训练动态响应太慢。β2=0.999 意味着二阶动量 v 的有效记忆窗口约 1000 步（1/(1-0.999)），即 v 是最近约 1000 步梯度的平均。大模型训练中梯度分布会在训练过程中变化（如不同阶段学的模式不同），0.999 的长记忆导致 v 滞后于当前梯度特性，自适应步长不够"实时"。β2=0.95 把记忆窗口缩到约 20 步，v 更紧跟当前梯度，自适应更灵敏。GPT-3/LLaMA 等用 0.95 是经验调出来的——在数百亿参数规模下，0.95 的收敛比 0.999 快且稳定。代价是 0.95 的 v 噪声更大（样本少），但配合 warmup 和 gradient clipping 能控制。所以 β2 不是"越大越稳"，是"匹配训练动态的响应速度"。小模型用 0.999（梯度分布稳定，长记忆无害），大模型用 0.95（梯度分布变化快，要灵敏响应）。

### 第四层：方案权衡

**Q：SGD 的泛化比 Adam/AdamW 好（找到 flat minima），你承认这点。但大模型仍用 AdamW 而非 SGD。如果"泛化好"是目标，为什么放弃 SGD？**

因为大模型训练的约束和传统 CV 不同。一是参数规模——SGD 要为百亿参数调学习率（可能要分层设不同 lr），调参成本极高且不一定收敛；AdamW 的自适应让一组默认超参（lr=3e-4, betas=(0.9,0.95)）能 work，省调参。二是训练成本——大模型训练动辄百万 GPU 小时，SGD 收敛慢 2-3 倍意味着成本翻倍，不可接受。三是数据规模——大模型用海量数据，过拟合风险相对小（数据 >> 参数），泛化优势的边际收益下降，而 AdamW 的收敛速度收益更关键。四是实证——LLaMA、GPT 等都用 AdamW 训出了泛化良好的模型，证明"AdamW + 海量数据 + 适当正则"的泛化够用。SGD 的 flat minima 优势在"小数据强正则"场景明显，大模型是"大数据"场景，权衡改变了。所以不是"放弃泛化"，是"泛化和成本的新权衡下 AdamW 更优"。

**Q：现在有 Lion、Shampoo、Adafactor 等新优化器，有的显存更省（Adafactor 不存完整 v）。为什么大模型仍主流用 AdamW，不换这些更新的？**

因为 AdamW 的"效果稳定 + 生态成熟"优势压过新优化器的"理论优势"。新优化器的问题：一是可复现性差——Lion（Google 2023）在某些任务好，但换数据集/模型架构时效果波动大，不如 AdamW 稳健；二是生态支持——训练框架（Megatron/DeepSpeed）、混合精度、ZeRO 分片都默认优化 AdamW，换优化器要改大量基建且可能有隐性 bug；三是"省显存"的收益被其他优化抵消——Adafactor 省的是 v 的显存（约参数量一半），但 ZeRO/FSDP 已经把优化器状态分片到多卡，单卡显存压力已缓解，Adafactor 的边际收益不大；四是调参成本——新优化器的超参（如 Lion 的 lr 要比 AdamW 小 10 倍）要重新调，大模型调参成本极高，社区倾向于用"默认就能 work"的 AdamW。所以 AdamW 是"足够好 + 生态绑定"的赢家，新优化器要在"显著优势 + 充分验证 + 生态支持"后才会被替代，这个周期通常几年。

### 第五层：验证与沉淀

**Q：你怎么证明 AdamW 比另一个优化器（如 SGD 或 Lion）在"你的任务"上更优？不是只看论文，而是你自己验证过的？**

在自己的任务上做对照实验。控制变量：同模型架构、同数据、同 batch size、同训练步数，只换优化器。比三类指标：一是收敛曲线（loss vs steps），看谁收敛快、最终 loss 低；二是下游任务效果（如 GLUE benchmark、ImageNet accuracy），看泛化；三是训练稳定性（多次独立运行的 loss 方差，梯度范数是否稳定）。如果 AdamW 在收敛速度和下游效果上都优或持平，且稳定性好，就用 AdamW。还要做超参敏感性测试——对每个优化器在其推荐超参附近网格搜索（lr 扫几个量级），看最优表现，避免"优化器 A 用了差超参所以输了"的不公平对比。证明逻辑是"同条件下多指标对比 + 超参调优后的最优 vs 最优 + 多次运行稳定性"，用自己任务的数字而非论文数字。

**Q：怎么让团队在大模型训练时用一致的优化器配置（AdamW + warmup + cosine + clipping），而不是各实验各试各的导致不可复现？**

把训练配置模板化和版本化。一是标准配置模板：团队维护一套"优化器 + 学习率调度 + 正则"的基线配置（如 AdamW lr=3e-4 betas=(0.9,0.95) wd=0.1 + warmup 2000 + cosine + clip 1.0），新实验默认用模板，偏离要记录理由。二是超参搜索规范：要调超参时用固定的搜索协议（如 lr 扫 1e-5 到 1e-3 对数均匀采 5 个点），报告所有结果而非只报最好的，避免"挑了好结果报"的偏差。三是实验日志：每次训练记录完整配置（优化器、超参、随机种子、数据版本、框架版本）到实验追踪系统（如 W&B/MLflow），保证可复现。四是回归基线：固定一个"基线实验"（如 LLaMA-7B 在标准数据上），每次框架/配置变更后重跑基线确认效果不退化，退化则定位是配置问题还是框架 bug。让训练是"可复现的实验科学"，靠模板 + 日志 + 回归，不靠个人经验。

## 结构化回答

**30 秒电梯演讲：** SGD是用"当前坡度"决定走哪一步，Adam是用"历史坡度的滑动平均+自适应步长"决定走哪一步。本质区别是Adam引入了"动量(惯性)"和"自适应学习率(每参数不同步长)"。

**展开框架：**
1. **SGD** — θ←θ−η∇L，仅用当前梯度
2. **Momentum** — θ←θ−η·v，v=βv+(1-β)∇L，引入惯性
3. **Adam** — 结合一阶动量(方向)和二阶动量(自适应步长)

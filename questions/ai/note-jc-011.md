# 【阶跃星辰面经】怎么理解时间复杂度

## 一、时间复杂度是什么

**时间复杂度**用 Big-O 记号描述**算法运行时间（操作次数）随输入规模 n 增长的上界**，忽略常数和低阶项，只看**增长趋势**。

```
O(f(n)) 表示：当 n → ∞ 时，操作次数 ≤ C · f(n)（C 是常数）

例如某算法操作次数 = 3n² + 5n + 100
  → 时间复杂度 O(n²)（忽略 3、5n、100）
```

**关键**：O 描述"增长趋势"，不是"具体多少秒"。O(n) 的算法在差机器上可能比 O(n²) 在好机器上慢（n 小时），但 n 够大时 O(n) 必胜。

## 二、常见复杂度阶（从快到慢）

```
O(1)       常数        数组下标访问、哈希表查询
O(log n)   对数        二分查找、平衡树操作
O(n)       线性        遍历数组
O(n log n) 线性对数    排序（快排/归并）、FFT
O(n²)      平方        冒泡/选择排序、矩阵乘法
O(2^n)     指数        暴力求子集、朴素斐波那契递归
O(n!)      阶乘        全排列、旅行商暴力
```

**增长速度对比**（n=1000）：
| 复杂度 | 操作次数 | 耗时 |
|--------|---------|------|
| O(log n) | 10 | 纳秒 |
| O(n) | 1000 | 微秒 |
| O(n log n) | 10000 | 微秒 |
| O(n²) | 100万 | 毫秒 |
| O(2^n) | 10^301 | 宇宙寿命 |
| O(n!) | 天文数字 | 永远 |

## 三、怎么评估时间复杂度

### 方法1：数循环嵌套

```python
# O(n) - 单层循环遍历
for i in range(n):
    process(i)

# O(n²) - 两层嵌套
for i in range(n):
    for j in range(n):
        process(i, j)

# O(n³) - 三层嵌套（矩阵乘法）
for i in range(n):
    for j in range(n):
        for k in range(n):
            c[i][j] += a[i][k] * b[k][j]

# O(log n) - 每次减半（二分查找）
while low <= high:
    mid = (low + high) // 2
    if arr[mid] == target: return mid
    elif arr[mid] < target: low = mid + 1
    else: high = mid - 1
```

### 方法2：递归用主定理（Master Theorem）

```
T(n) = a·T(n/b) + f(n)
  a：递归调用次数
  n/b：每次子问题规模
  f(n)：合并代价

例：归并排序 T(n) = 2T(n/2) + O(n) → O(n log n)
    二分查找 T(n) = T(n/2) + O(1) → O(log n)
```

## 四、忽略常数和低阶项

```python
def f(n):
    for i in range(1000000):    # 常数，忽略
        pass
    for i in range(n):          # O(n)
        process(i)
    for i in range(n):          # O(n)
        for j in range(n):      # O(n²)
            process(i, j)
# 总：O(1) + O(n) + O(n²) = O(n²)（取最高阶）
```

**为什么忽略常数**：Big-O 看 n→∞ 的趋势，常数在 n 够大时被高阶项主导。

## 五、实际判断：算法能不能过

```
竞赛/面试经验（1秒约 10^8 次操作）：
  O(n)     n 可到 10^8
  O(n log n) n 可到 10^6
  O(n²)    n 可到 10^4
  O(n³)    n 可到 500
  O(2^n)   n 只能 20-25
  O(n!)    n 只能 10

例：n=10^5 的排序题 → 必须 O(n log n)，O(n²) 会 TLE
```

## 六、加分点

- 说出 **O 是上界，Ω 是下界，Θ 是确界**（紧致）
- 说出 **排序的 O(n log n) 下界**：基于比较的排序最快 O(n log n)
- 说出 **均摊复杂度**：如动态数组 append 平均 O(1)（偶尔扩容 O(n) 但均摊 O(1)）

## 七、雷区

- ❌ "O(2n) 和 O(n) 不同" → 相同（常数忽略）
- ❌ 只看嵌套层数不看每次操作 → 如循环内调用 O(n) 函数
- ❌ 混淆最好/最坏/平均复杂度 → Big-O 通常指最坏

## 八、扩展

- **空间复杂度**：类似时间，描述额外内存随 n 增长（如递归栈深度）
- **最好/最坏/平均**：快排最好 O(n log n)，最坏 O(n²)，平均 O(n log n)
- **Big-O 的严格定义**：O(g(n)) = {f(n) | ∃c,n0 使 0≤f(n)≤c·g(n) ∀n≥n0}

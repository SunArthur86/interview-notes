# 【阶跃星辰面经】bit 和 byte 之间的对应关系

## 一、基本单位

| 单位 | 含义 | 换算 |
|------|------|------|
| **bit（位）** | 计算机最小单位（0 或 1） | - |
| **byte（字节）** | 基本存储单位 | **1 byte = 8 bit** |

## 二、为什么 1 byte = 8 bit

```
历史：1964 年 IBM System/360 确立 8 位字节标准

原因：
  - 8 bit 能表示 2^8 = 256 种值
  - 足够编码 ASCII 字符（128 个字符 + 扩展）
  - 2^8 是 2 的幂，硬件实现方便（地址线对齐）
  - 比 7 bit 多 1 位做校验，比 6 bit 能表示更多字符

8 bit byte 成为标准后，所有现代计算机沿用。
```

## 三、衍生单位（1024 进制）

```
1 byte (B)   = 8 bit
1 KB         = 1024 B      = 2^10 B
1 MB         = 1024 KB     = 2^20 B
1 GB         = 1024 MB     = 2^30 B
1 TB         = 1024 GB     = 2^40 B
1 PB         = 1024 TB     = 2^50 B
```

**为什么 1024 不是 1000**：计算机用二进制，2^10=1024 是最接近 1000 的 2 的幂。

## 四、常见数据类型的字节数

| 类型（C 语言） | 字节数 | bit 数 | 范围 |
|--------------|--------|--------|------|
| char | 1 | 8 | -128~127 或 0~255 |
| short | 2 | 16 | -32768~32767 |
| int | 4 | 32 | -2^31~2^31-1 |
| long (64位) | 8 | 64 | -2^63~2^63-1 |
| float | 4 | 32 | 单精度浮点 |
| double | 8 | 64 | 双精度浮点 |
| 指针(64位) | 8 | 64 | 地址 |

## 五、实际应用：算模型显存

```
7B 参数模型，FP16 存储：
  参数量 = 7 × 10^9
  每参数 2 byte（FP16）
  显存 = 7e9 × 2 = 14 GB

加 Adam 优化器状态（FP32，参数的2倍）：
  = 7e9 × 2 (梯度FP16) + 7e9 × 4 (优化器m) + 7e9 × 4 (优化器v)
  = 14 + 28 + 28 = 70 GB
  + 参数本身 14GB = 84 GB（接近 A100 80GB 上限）
```

## 六、bit/byte 易混淆点

```
网络带宽：100 Mbps = 100 megabit/s = 12.5 MB/s（下载速度）
  → 注意是 bit 不是 byte，除以 8 才是实际速度

存储容量：硬盘 1 TB = 10^12 B（厂商用1000进制）
  → 操作系统显示 931 GB（用1024进制）→ "缩水"错觉
```

## 七、加分点

- 说出 **1 byte = 8 bit 是 IBM System/360 确立的标准**
- 说出 **网络带宽用 bit（Mbps），存储用 byte（MB）** 的区别
- 说出 **1024 进制源于二进制**（2^10=1024）

## 八、雷区

- ❌ "1 KB = 1000 B" → 是 1024（二进制），但硬盘厂商用 1000（营销）
- ❌ 混淆 Mbps 和 MBps → 差 8 倍
- ❌ "1 byte = 4 bit" → 是 8

## 九、扩展

- **KiB vs KB**：IEC 标准 KiB=1024，KB=1000（严格区分），但日常混用
- **汉字编码**：UTF-8 一个汉字 3 字节，GBK 一个汉字 2 字节
- **INT8 量化**：把 FP32 参数转成 8 位整数，显存省 4 倍（推理加速）

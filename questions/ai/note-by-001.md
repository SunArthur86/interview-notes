# 【字节面经】结合 Agent 思想，如何重构传统 AI 任务（数据标注、模型训练、质检）的完整链路？

## 一、传统 AI 任务链路的痛点

```
传统链路：
  数据采集 → 人工标注 → 模型训练 → 人工质检 → 上线

痛点：
  1. 人工标注：慢、贵、质量不一致（标注员水平参差）
  2. 模型训练：调参靠经验、消融实验靠人工跑、失败分析靠猜
  3. 人工质检：只能抽检（5-10%），漏掉大量 bad case
  → 整条链路被"需要判断力但重复"的人工环节拖累
```

## 二、Agent 重构：三环节自动化

### 环节1：数据标注——多 Agent 协同

```
传统：标注员看一条数据 → 判断 → 标注（慢、不一致）

Agent 重构（多 Agent 协同）：
  ┌─────────────┐
  │ 抽取 Agent   │  ← LLM 理解数据，初标注
  └──────┬──────┘
         │
  ┌──────▼──────┐
  │ 校验 Agent   │  ← 规则 + LLM 检查标注合理性
  └──────┬──────┘
         │ 有争议？
  ┌──────▼──────┐
  │ 仲裁 Agent   │  ← 多模型投票或更强模型裁决
  └──────┬──────┘
         │ 仍不确定？
  ┌──────▼──────┐
  │ 人工兜底     │  ← 只处理 Agent 搞不定的
  └─────────────┘
```

**关键设计**：
- **置信度路由**：Agent 标注带置信度，低置信度转人工
- **多 Agent 投票**：多个模型独立标注，一致则采纳，分歧则仲裁
- **主动学习**：挑出 Agent 最不确定的样本给人工标（高信息量）

**效果**：80% 数据 Agent 自动标注，20% 人工兜底，效率提升 5 倍。

### 环节2：模型训练——Agent 自动化实验

```
传统：研究员手动调参 → 跑实验 → 看结果 → 再调（慢、靠经验）

Agent 重构（实验编排 Agent）：
  ┌──────────────────┐
  │ 实验规划 Agent    │  ← 根据目标生成实验方案（超参组合）
  └───────┬──────────┘
          │
  ┌───────▼──────────┐
  │ 训练执行 Agent    │  ← 自动跑训练、监控指标、记录日志
  └───────┬──────────┘
          │
  ┌───────▼──────────┐
  │ 结果分析 Agent    │  ← 分析实验结果、对比 baseline、归因
  └───────┬──────────┘
          │
  ┌───────▼──────────┐
  │ 迭代决策 Agent    │  ← 决定下一步实验（贝叶斯优化/启发式）
  └──────────────────┘
```

**应用场景**：
- **超参搜索**：Agent 用贝叶斯优化自动搜索学习率/batch size
- **消融实验**：Agent 自动设计消融组合、跑实验、对比
- **失败分析**：训练发散时 Agent 分析日志（lr 太大？数据有问题？）

**效果**：研究员从"手动跑实验"变成"review Agent 的实验报告"。

### 环节3：质检——LLM-as-Judge Agent

```
传统：质检员抽检 5-10%，大量 bad case 漏网

Agent 重构：
  质检 Agent（LLM-as-Judge）→ 全量自动质检
    │
    ├─ 规则检查：格式/字段/数值范围（确定性）
    ├─ LLM 判断：语义合理性/边界 case（判断力）
    └─ 多 Agent 投票：重要 case 用多模型交叉验证

  质检结果：
    ├─ 通过：放行
    ├─ 可疑：转人工复核
    └─ 明确错误：自动修正或退回
```

**LLM-as-Judge 的关键**：
- **校准**：用人工标注的 golden set 校准 Judge 偏差
- **多 Judge 投票**：避免单模型偏好
- **置信度路由**：Judge 不确定的转人工

**效果**：从 5% 抽检变成 100% 全检，bad case 发现率提升 10 倍。

## 四、关键设计原则

### 1. Agent 处理"判断力 + 重复"，人工处理"边界 case"
```
Agent 擅长：大量重复性判断（标注/质检/调参）
人工擅长：Agent 搞不定的疑难（歧义数据/创新实验/重要决策）
→ 用置信度路由分配：高置信度 Agent 做，低置信度人工做
```

### 2. 数据飞轮（闭环）
```
Agent 标注的数据 → 训练更好的模型 → 更好的 Agent → 更好的标注
质检发现的 bad case → 补充到训练集 → 模型改进 → bad case 减少
→ 形成"越用越好"的飞轮
```

### 3. 多 Agent 协同 + 仲裁
```
单 Agent 有偏差 → 多 Agent 投票/仲裁
  - 标注：3 个模型独立标，多数投票
  - 质检：2 个 Judge 交叉验证
  - 分歧：更强模型或人工仲裁
```

## 五、加分点

- 说出 **不是消灭人工，而是"人工只做 Agent 做不了的边界 case"**
- 说出 **数据飞轮**：Agent 产出反哺训练，越用越好
- 说出 **置信度路由**是"哪些给 Agent、哪些给人工"的关键机制

## 六、雷区

- ❌ "Agent 完全替代人工" → 边界 case 仍需人工，关键是置信度路由
- ❌ "每个环节一个 Agent 就行" → 复杂判断需多 Agent 协同+仲裁
- ❌ 忽略数据飞轮 → 没有闭环，Agent 不会自我改进

## 七、扩展

- **RLAIF（AI Feedback）**：用 Agent 的判断代替人工反馈做 RLHF，降低人工成本
- **Self-Play 数据生成**：Agent 自己生成训练数据（如对抗样本）
- **Active Learning**：Agent 主动挑"最有信息量"的样本给人工标，最大化标注效率

# 【阿里淘天AI二面】如何避免Agent场景下模型的幻觉问题？

> 来源：阿里巴巴淘天淘工厂 AI应用开发 二面面经（小红书）

## 一、幻觉的本质与根源

### 🎯 什么是幻觉

```
幻觉 = LLM生成"看似合理但与事实不符"的内容

示例：
  用户：阿里巴巴是哪年成立的？
  模型（幻觉）：阿里巴巴成立于1998年3月。  ← 错误，实际是1999年

  模型不知道"不知道" → 总是生成最可能的回答
  这是概率生成模型的固有特性
```

### 幻觉的类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **事实性幻觉** | 生成与客观事实矛盾的内容 | "太阳是第9大行星" |
| **忠实性幻觉** | 输出与给定上下文/文档矛盾 | 文档说A，模型回答B |
| **推理幻觉** | 推理步骤逻辑错误 | 前提正确但推导过程有误 |
| **来源幻觉** | 编造不存在的引用/来源 | "根据Smith et al. (2023)..." |

### 为什么会产生幻觉

```
LLM本质：next-token prediction
  P(y | x) = 最大化给定上文x，下一个token y的概率

  当模型内部知识不充分时：
  ┌─────────────────────────────────┐
  │ 知识库中没有"阿里巴巴成立年份"     │
  │ 但训练数据中见过类似公司信息       │
  │ → 模型用概率推断"编造"一个答案    │
  │ → "1998年"（虽然错误，但概率最高） │
  └─────────────────────────────────┘

  核心原因：
  1. 训练数据噪声/过时
  2. 模型概率推断产生的"创造"
  3. 缺乏事实校验机制
  4. 没有显式的"不确定"输出机制
```

## 二、三层防线

### 第一层：提示词角度（预防）

```
策略1：约束模型基于知识库回答
  ┌─────────────────────────────────────────────┐
  │ System Prompt:                               │
  │                                              │
  │ 你是一个智能客服Agent。你必须：               │
  │ 1. 只基于提供的知识库文档回答问题             │
  │ 2. 如果知识库中没有相关信息，回答              │
  │    "抱歉，我暂时无法回答这个问题，            │
  │     正在为您转接人工客服"                     │
  │ 3. 不要编造、猜测或推断知识库中没有的信息      │
  │ 4. 每个回答必须标注信息来源                   │
  └─────────────────────────────────────────────┘

策略2：要求标注来源
  "请在回答末尾标注信息来源，格式：[来源：文档名-段落]"

策略3：设置"不知道"兜底
  "当你不确定答案时，必须回答'我不确定'而不是编造答案"

策略4：降低温度参数
  temperature = 0.1 ~ 0.3
  低温度 → 减少随机性 → 减少创造性幻觉
```

### 第二层：Workflow角度（过程控制）

```
策略1：结构化决策输出

  强制Agent输出JSON Schema：
  ┌─────────────────────────────────────────┐
  │ {                                       │
  │   "answer": "阿里巴巴成立于1999年",      │
  │   "source": "公司简介.pdf - 第3页",      │
  │   "confidence": 0.95,                   │
  │   "needs_human": false                  │
  │ }                                       │
  └─────────────────────────────────────────┘

  校验规则：
  - source为空 → 拒绝输出（无依据的回答）
  - confidence < 0.7 → 转人工
  - answer中包含source中没有的信息 → 拒绝输出

策略2：分步决策，每步可校验

  Step 1: 意图识别 → 输出intent + confidence
  Step 2: 知识检索 → 输出documents + relevance_score
  Step 3: 答案生成 → 输出answer + source + confidence
  Step 4: 答案校验 → 输出pass/fail + reason

  每步都有明确的输出Schema和校验规则
  → 任何一步校验失败，不会传递到下一步

策略3：RAG检索增强

  用户问题 → 向量检索Top-K文档 → Rerank精排
  → LLM基于检索结果生成答案（而非依赖内部知识）

  关键：答案必须引用检索到的文档内容
```

```python
from pydantic import BaseModel, Field

class AgentResponse(BaseModel):
    """结构化输出Schema"""
    answer: str = Field(description="回答内容，必须基于retrieved_docs")
    source: str = Field(description="信息来源文档名和段落")
    confidence: float = Field(ge=0, le=1, description="置信度")
    needs_human: bool = Field(description="是否需要转人工")

    def validate_against_docs(self, retrieved_docs: list[str]) -> bool:
        """校验answer中的关键信息是否在retrieved_docs中"""
        if not self.source:
            return False
        if self.confidence < 0.7:
            return False
        # 可选：NLI模型检查answer与docs的一致性
        return True
```

### 第三层：后处理角度（兜底校验）

```
策略1：双模型校验

  模型A（生成模型）→ 生成答案
       │
       ▼
  模型B（审核模型）→ 校验答案是否准确安全
       │
       ├── 通过 → 输出给用户
       └── 不通过 → 重新生成 OR 转人工

  为什么有效？
  - 不同模型的幻觉模式不同
  - 两个模型同时产生相同幻觉的概率很低
  - 审核模型可以基于检索文档做事实核查

策略2：自一致性投票（Self-Consistency）

  同一个问题，让模型生成N个答案（temperature=0.7）：
  答案1: "1999年"
  答案2: "1999年"
  答案3: "1998年"
  答案4: "1999年"
  答案5: "1999年"

  → 多数投票：1999年（4/5一致）

  幻觉是随机的，多次生成中幻觉内容通常不一致
  → 多数投票可以过滤掉随机幻觉

策略3：置信度过滤

  对Agent输出做置信度评估：
  - 基于logits的概率（如果可用）
  - 基于生成模型的self-evaluation
  - 基于检索结果的相关性分数

  confidence < threshold → 拒绝输出 / 转人工
```

## 四、面试加分点

1. **提到RAG幻觉的根因**：检索质量差（召回不相关文档）会导致模型基于错误信息生成答案，所以rerank很重要
2. **提到Grounding技术**：Google的Grounding API，将模型输出与搜索结果做实时事实验证
3. **提到Constitutional AI**：Anthropic的方法，让AI自己根据"宪法"（原则）来审查和修正自己的输出
4. **提到量化幻觉率**：使用RAGAS、TruLens等框架量化幻觉率（faithfulness score）
5. **提到温度与幻觉的关系**：temperature=0能显著减少随机性幻觉，但不能完全消除（模型本身的参数化知识可能就是错的）

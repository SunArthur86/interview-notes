# 【字节面经】手撕：不大于 n 的最大值

## 一、先澄清题意（重要考点）

这类题面略含糊，**第一反应应该是和面试官确认**——这本身就是考察点。常见两种解读：

**解读1**：给定一组数字（如 `{1, 2, 4, 9}`），用这些数字（可重复）拼出 **≤ n 的最大数**。
> 例：digits={1,2,4,9}, n=2345 → 拼 "2299"？不对，最大是 "2249"？需算。

**解读2**：找 **≤ n 的最大自幂数/完全平方数/满足某性质的数**。
> 例：≤ n 的最大完全平方数 → floor(√n)²。

以下按**解读1**（更常见、更有算法含量）展开。

## 二、解读1：用给定数字拼 ≤ n 的最大数

### 思路：贪心 + 回溯

```
从高位到低位：
  1. 尽量填能填的最大数字（贪心）
  2. 如果某位填了 X 后，剩余位即使全填最大也不够/超出，回溯试更小数字
  3. 特殊情况：n 是 3 位数，但用给定数字拼 3 位必超 → 拼位数少一位的全最大数
```

### 完整实现

```python
def largestNumberNotGreaterThanN(digits: list[int], n: int) -> str:
    """
    用 digits 中的数字（可重复）拼出 ≤ n 的最大数。
    digits 如 [1, 2, 4, 9]
    """
    digits = sorted(set(digits))
    s = str(n)
    length = len(s)
    
    def backtrack(pos, is_tight, current):
        """
        pos: 当前填到第几位
        is_tight: 前面是否和 n 完全一致（True 表示受 n 约束）
        current: 已填的数字列表
        返回: 能拼出的最大数字字符串，或 None（拼不出）
        """
        if pos == length:
            return ''.join(current) if current else None
        
        # 这一位的上限
        upper = int(s[pos]) if is_tight else 9
        
        # 从大到小试（贪心）
        for d in reversed(digits):
            if d > upper:
                continue  # 超过 n 这一位，跳过
            # 填 d
            current.append(str(d))
            result = backtrack(
                pos + 1,
                is_tight and (d == upper),
                current
            )
            if result is not None:
                return result  # 找到最大的，直接返回
            current.pop()
        
        return None  # 这一位填什么都不行
    
    result = backtrack(0, True, [])
    
    # 如果位数和 n 相同拼不出（如 n=200, digits=[3,4,5] 都>2）
    # 退一位拼全最大
    if result is None and length > 1:
        # 拼位数少一位的、全用最大数字
        result = str(max(digits)) * (length - 1)
        # 去掉前导 0（如果有）
        result = result.lstrip('0') or '0'
    
    return result if result else "0"
```

### 示例

```python
# 例1: digits=[1,2,9], n=2345
# 第0位: 上限2，试9(超)→试2(填)，is_tight=True
# 第1位: 上限3，试9(超)→试2(填)，is_tight=False（2<3）
# 第2位: is_tight=False，上限9，填9
# 第3位: 填9
# 结果: "2299"

# 例2: digits=[5,6], n=100  
# 第0位: 上限1，试6(超)→试5(超)→None

```

## 三、简化版（数字可重复，常见面试版）

```python
def largest_smaller_or_equal(digits, n):
    """简化: 贪心从高位填，每位填 ≤ 对应位的最大可用数字"""
    digits = sorted(digits, reverse=True)
    s = str(n)
    
    # 尝试 1 到 len(s) 位
    for length in range(len(s), 0, -1):
        # 尝试用 length 位拼最大数
        if length < len(s):
            # 位数少，直接全填最大
            return str(digits[0]) * length
        
        # 位数相同，受 n 约束
        result = []
        for i, ch in enumerate(s):
            upper = int(ch)
            # 找 ≤ upper 的最大可用数字
            placed = False
            for d in digits:
                if d <= upper:
                    result.append(str(d))
                    placed = True
                    break
            if not placed:
                break  # 这位填不了
        else:
            # 成功填完 length 位
            return ''.join(result)
    
    return "0"  # 都不行
```

## 四、解读2：≤ n 的最大满足某性质的数

### 例：≤ n 的最大完全平方数

```python
import math
def largest_square_leq(n):
    return math.isqrt(n) ** 2  # math.isqrt 是整数平方根
```

### 例：≤ n 的最大 2 的幂

```python
def largest_power_of_2_leq(n):
    if n <= 0: return 0
    # 找最高位的 1
    return 1 << (n.bit_length() - 1)
```

## 五、加分点

- 说出 **先和面试官澄清题意**（数字拼数 vs 范围找性质）
- 说出 **高位贪心**：数值大小由高位决定，高位尽量填大
- 说出 **回溯**：当前选择导致拼不出/超出就回退

## 六、雷区

- ❌ 不澄清题意直接写 → 可能答错方向
- ❌ 高位填不下时不回溯 → 漏解
- ❌ 忘记处理"位数少一位"的情况 → n=200 digits=[3,4] 应返回 "44"

## 七、扩展

- **数位 DP**：此类"≤ n 的满足条件数"通用解法，用记忆化搜索
- **前导 0 处理**：如拼 3 位数时首位不能是 0
- **数字不可重复**：需要额外记录已用数字（全排列变体）

## 记忆要点

- 核心策略：贪心+回溯，从高位到低位尽量填能填的最大数字

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：这道题"不大于 n 的最大值"题面含糊，你为什么不直接写代码，要先和面试官澄清题意？**

因为这题至少有两种解读，写错方向是 0 分。解读 1：给定数字集合（如 {1,2,4,9}），用这些数字拼出不超过 n 的最大数（贪心+回溯）。解读 2：从某范围找不超过 n 的满足某条件的最大值（如"各位不递减的最大数"，贪心或二分）。两种解法完全不同——解读 1 是"约束下的数位构造"，解读 2 是"约束下的搜索"。先澄清体现两点能力：1）工程素养——遇到歧义需求先澄清是工程师的本能；2）沟通能力——面试官就是未来的同事，能高效对齐需求是关键软技能。直接写代码等于赌题意，赌错浪费面试时间还暴露沟通短板。

### 第二层：证据与定位

**Q：假设澄清后是解读 1（用 {1,2,4,9} 拼不超过 n 的最大数），你怎么验证你的贪心+回溯思路是对的，不会漏解？**

用反证法验证贪心的正确性。贪心策略：从最高位到最低位，每位尽量填最大的可用数字，如果填了之后"剩余位数即使全填最小数字也超过 n"，就回溯换成更小的数字。正确性论证：数值大小由高位决定（高位的 1 大于所有低位的 9），所以高位优先填最大数字是最优的——如果某位可以填 9 但你填了 4，无论低位怎么填都小于"填 9 的最大可能"，因此贪心选择不影响全局最优。回溯是兜底——当高位填最大导致低位"无解"（即使全填最小也超 n）时，回退高位换次大数字。这套"高位贪心 + 无解回溯"保证找到合法的最大数。

### 第三层：根因深挖

**Q：贪心+回溯里，什么情况下会触发回溯？能不能举一个具体例子说明什么时候贪心会失败？**

举例：n=4321，数字集合 {1,2,4,9}。贪心：最高位填 9？但 9xxx 任何数都 >4321（千位 9>4），不行。填 4？4xxx 可能可以（4=4，看低位）。第二位填 9？49xx，但 49xx>43xx（百位 9>3），不行。填 3？但 3 不在集合 {1,2,4,9} 里。填 2？42xx，4=4,2<3，合法，低位全填最大 9→4299。所以答案是 4299。这里回溯发生在：第一位 9 不行（直接超），第二位 9 不行（超 n 的对应位），要逐个降级试。最坏情况每位都要试到最小数字才找到合法解，但总搜索空间是 O(位数 × 数字集大小)，远小于暴力枚举 O(数字集^位数)。回溯的剪枝（"填了这个高位后低位无解就别往下试"）是效率关键。

**Q：如果数字集合里有 0，或者 n 的位数和集合能拼出的位数不同，边界怎么处理？**

三类边界：1）含 0——0 不能做首位（如 0123 不是合法 4 位数），贪心时首位跳过 0，后续位可以用 0；2）拼出的位数 < n 的位数——如 n=4321（4 位），集合能拼出 3 位数，那 3 位的最大数（如 999）一定 <n 的任何 4 位数，直接取集合最大数字填满 3 位；3）拼出的位数 > n 的位数——不可能，因为位数多一定更大（超 n），不可能合法。实务中先比较"能拼的最大位数"和"n 的位数"：若 <，直接全填最大数字；若 =，走贪心+回溯；若 >（理论上不会，因为会超 n）。这些边界要在代码开头用 if 显式处理，否则贪心主体会出错。

### 第四层：方案权衡

**Q：这道题贪心+回溯是 O(位数 × 数字集大小)，为什么不用动态规划或记忆化搜索？**

贪心+回溯已经是最优且最直观。DP 的状态定义会很别扭——"dp[位][是否已小于 n][已用数字集合]"，状态数 O(位数 × 2 × 2^数字集大小)，当数字集大时（如 10 个数字）状态爆炸（2^10=1024）。而且这道题的贪心有严格的数学正确性（高位决定大小），不需要 DP 来"穷举所有可能找最优"——贪心直接构造最优，DP 是杀鸡用牛刀。回溯只在贪心的某步"走不通"时局部回退，不是全局搜索，效率远高于 DP。所以这题贪心+回溯是"最合适"而非"最快"——它匹配了问题的"高位贪心"结构，DP 反而绕远路。

**Q：如果题面变成解读 2（如"找不超过 n 的、各位数字不递减的最大数"），思路完全变了，你怎么快速切换解法？**

解读 2 的约束是"数字的性质"而非"用给定数字拼"，解法切换为"从 n 往下找"或"贪心构造"。如"各位不递减的最大数 ≤n"：从 n 的最高位开始，如果遇到"递减点"（如 n=4321 的 3<4），把这位减 1（变 3）、后面全填 9（最大且满足不递减，因为 3<9 不违反）→3399...不对，3<3 不严格。更优：从高位往低位，维护"不递减"，一旦破坏就把当前位降一位、后续全填 9，处理借位。复杂度 O(位数)。关键区别：解读 1 是"给定集合的数位构造"（回溯），解读 2 是"数字性质的贪心构造"（从 n 往下或贪心调整）。面试时先确认是"集合约束"还是"性质约束"，直接定解法方向。

### 第五层：验证与沉淀

**Q：你怎么验证贪心+回溯的实现是正确的，尤其在边界 case？**

三层验证：1）手算 case——选 n=4321, 集合{1,2,4,9}，手算答案 4299，代码输出对比；2）暴力对照——数字集小时，写一个暴力枚举（全排列所有可能拼接，过滤≤n 取最大），随机生成 100 个 (n, 集合) 对比贪心输出，全一致；3）边界专项——空集合、集合只含 0、n 是单 digit、n 本身能用集合拼出（答案=n）等极端 case，每个单独测。贪心+回溯的常见 bug：忘记处理首位不能是 0、回溯时没正确恢复状态（如 used 标记没清）、位数判断错误（拼出位数<n 位数时没走"全填最大"分支）。这些边界用专项 case 覆盖，暴力对照兜底。

**Q：这类"数位构造/约束最值"题的解题经验怎么沉淀成面试肌肉记忆？**

总结成"数位题三步法"：1）先澄清——题面含糊时必问"是集合拼数还是性质约束""首位能否为 0""数字能否重复用"；2）定结构——集合拼数→贪心（高位填最大）+回溯（无解降级）；性质约束→从 n 贪心调整或二分；位数比较→分情况（拼出位数 vs n 位数）；3）验证——贪心正确性用反证法论证（"高位选小的一定不优"），代码用暴力对照验证。把这三步和"无重复子串的滑动窗口"等模板并列，形成"题型 → 解法模板"的映射表，面试看到数位题直接走三步法，避免临场纠结解法。

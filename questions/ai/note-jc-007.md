# 【阶跃星辰面经】Adam 优化器的公式：一阶矩、二阶矩、梯度更新

## 一、Adam 完整公式

```
给定：参数 θ，学习率 lr，梯度 g_t，超参 β1=0.9, β2=0.999, ε=1e-8

[1] 一阶矩（动量）更新：
    m_t = β1 · m_{t-1} + (1 - β1) · g_t
    
[2] 二阶矩（梯度幅度）更新：
    v_t = β2 · v_{t-1} + (1 - β2) · g_t²

[3] 偏差修正（修正初始偏差）：
    m_hat = m_t / (1 - β1^t)
    v_hat = v_t / (1 - β2^t)

[4] 参数更新：
    θ_t = θ_{t-1} - lr · m_hat / (√v_hat + ε)
```

## 二、一阶矩 m_t：动量（方向）

```
m_t = β1 · m_{t-1} + (1 - β1) · g_t
```

**本质**：梯度的**指数移动平均（EMA）**，代表"历史梯度的方向"。

- β1=0.9：当前梯度占 10%，历史占 90% → 平滑
- 作用：**加速收敛 + 抗噪**（单步梯度噪声被历史平均稀释）

**直觉**：像球往下滚带惯性——即使某一步梯度方向变了（噪声），球的运动方向仍由历史主导。

## 三、二阶矩 v_t：梯度幅度

```
v_t = β2 · v_{t-1} + (1 - β2) · g_t²
```

**本质**：梯度**平方**的 EMA，代表"历史梯度的幅度"。

- β2=0.999：更平滑（历史占 99.9%）
- 作用：**自适应学习率**——梯度幅度大的参数，学习率自动调小（除以大√v）；幅度小的调大

**直觉**：陡的地方（梯度大）小步走（防跨过最优点），平的地方（梯度小）大步走（加速）。

## 四、为什么要除以 √v_hat（自适应学习率）

```
更新量 = lr · m_hat / √v_hat

  m_hat：方向（带历史平滑）
  √v_hat：历史梯度幅度

某参数历史梯度一直很大（如 g≈10）：
  → v_hat≈100, √v_hat≈10
  → 更新量 = lr · 10 / 10 = lr（小步）

某参数历史梯度一直很小（如 g≈0.1）：
  → v_hat≈0.01, √v_hat≈0.1
  → 更新量 = lr · 0.1 / 0.1 = lr（其实一样？）
```

**关键**：除以 √v_hat 让每个参数的有效学习率归一化——梯度大的参数步长不被其幅度主导，而是由"方向一致性"决定。

## 五、为什么需要偏差修正

```
初始化 m_0=0, v_0=0

第一步（t=1）：
  m_1 = β1·0 + (1-β1)·g_1 = 0.1·g_1   （严重低估！真实应是 g_1）
  v_1 = β2·0 + (1-β2)·g_1² = 0.001·g_1²

偏差修正：
  m_hat = m_1 / (1-β1^1) = 0.1g_1 / 0.1 = g_1  ✓（修正回真实值）
  v_hat = v_1 / (1-β2^1) = 0.001g_1² / 0.001 = g_1²  ✓
```

**原因**：初始 m_0=v_0=0，前几步的 EMA 严重偏向 0（被初始值拖累）。偏差修正除以 (1-β^t) 补偿这个初始偏差。t 增大时 β^t→0，修正项→1，自动失效。

## 六、Adam vs SGD vs Momentum vs RMSProp

| 优化器 | 用一阶矩 | 用二阶矩 | 特点 |
|--------|---------|---------|------|
| SGD | ❌ | ❌ | 只用当前梯度，噪声大 |
| Momentum | ✅ | ❌ | 加历史方向，加速 |
| RMSProp | ❌ | ✅ | 加历史幅度，自适应 |
| **Adam** | ✅ | ✅ | **两者结合，默认选择** |

## 七、Adam 的超参与 LLM 训练

```
默认值（CV 时代）：β1=0.9, β2=0.999, lr=1e-3
LLM 训练常用：    β1=0.9, β2=0.95, lr=6e-4（GPT-3）
                 β1=0.9, β2=0.999, lr=3e-4（更稳）

为什么 LLM 用更小 β2（0.95 vs 0.999）：
  β2 小 → v_t 更关注近期梯度 → 自适应更敏感
  → LLM 训练梯度分布变化快，需要更快适应
```

## 八、AdamW（Adam 的改进）

```
Adam 的 weight decay 实现有问题（和自适应学习率耦合）：
  L2 正则加在梯度上 → 被除以 √v_hat → 各参数 weight decay 不一致

AdamW 修正：weight decay 直接作用在参数上，不和梯度耦合
  θ = θ - lr · (m_hat/√v_hat + wd · θ)
  → 各参数 weight decay 一致
  → LLM 训练标准用 AdamW
```

## 九、加分点

- 说出 **Adam = Momentum + RMSProp**（一阶+二阶）
- 说出 **偏差修正的原因**：初始 m_0=v_0=0 导致前几步低估
- 说出 **AdamW 的改进**：decoupled weight decay

## 十、雷区

- ❌ "Adam 用二阶矩就是用二阶导数" → 是梯度平方的 EMA，不是二阶导
- ❌ 忽略偏差修正 → 不修正前几步训练会偏
- ❌ "Adam 永远比 SGD 好" → SGD+Momentum 泛化更好（某些 CV 任务）

## 十一、扩展

- **学习率 warmup**：LLM 训练前几百步线性增 lr，配合 Adam 防止初期 v_t 不稳导致发散
- **Lookahead**：Adam 的 wrapper，用"快慢两个优化器"提升泛化
- **Sophia / Lion**：更新的优化器，声称比 Adam 更适合 LLM

## 记忆要点

- 核心本质：Adam = Momentum(一阶动量) + RMSProp(二阶矩)
- 一阶矩控方向：梯度的EMA抗噪加速；二阶矩控步长：除以√v自适应学习率
- 偏差修正原因：初值设0导致前几步严重低估，需除以(1-β^t)补偿
- 超参记忆：LLM常用β2=0.95（非默认0.999）以更快适应梯度变化
- 扩展：AdamW解耦了weight decay和自适应学习率，是LLM标配

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：Adam 同时维护一阶矩 m 和二阶矩 v。但更新公式 θ -= lr·m/√v 里，m 和 v 都来自同一个梯度 g。为什么不直接用 g（当前梯度），而要绕一圈算 EMA？当前梯度不是最"新鲜"的信息吗？**

因为单步梯度噪声太大。LLM 训练的梯度来自 mini-batch（如 batch=4M tokens 的子集），不同 batch 的梯度方向和幅度差异大（数据分布不均、稀疏激活、长尾 token），单步梯度 g_t 是真实梯度的噪声估计。直接用 g_t 更新（SGD）会被噪声带偏，震荡严重。EMA（指数移动平均）的作用是"用历史多步的加权平均降噪"——m_t 是过去多步梯度的平滑（β1=0.9 意味着有效窗口约 10 步），v_t 是梯度幅度的平滑（β2=0.95-0.999 窗口更长）。平滑后的 m 方向更稳定（多步一致的方向被强化，随机噪声被抵消），v 幅度更准（反映该参数的长期梯度大小，而非单 batch 的偶然值）。"新鲜"和"稳定"是权衡——单步新鲜但噪声大，EMA 稍滞后但稳。LLM 训练动辄几十万步，用 EMA 换稳定性远比追"新鲜"重要（新鲜的单步梯度常是噪声）。且 v 的长期统计是"自适应学习率"的基础——知道某参数长期梯度大（如 embedding 热门词），调小它的学习率，这是单步 g 给不了的信息。

### 第二层：证据与定位

**Q：你说 EMA 降噪让训练更稳。但训练日志里怎么验证"EMA 确实降噪了"？怎么对比 Adam vs SGD 在同一任务的梯度质量差异？**

观测几路信号。一是梯度的平滑度：记录每步的 grad_norm（全模型梯度 L2 范数），SGD 的 grad_norm 序列抖动剧烈（batch 间噪声直接传导），Adam 的 effective grad_norm（即 m/√v 的范数）更平滑（EMA 降噪）。量化指标是 grad_norm 序列的变异系数（std/mean），SGD 应高于 Adam。二是 loss 曲线平滑度：SGD 的 loss 抖动大（梯度噪声→参数震荡→loss 波动），Adam 的 loss 更平滑。三是参数更新的方向一致性：记录相邻步的更新方向夹角（cosine similarity），SGD 相邻步方向可能相反（噪声导致震荡），Adam 的 m 方向更一致。四是收敛速度对比：同任务同数据，SGD 和 Adam 达到同一 loss 水平的 step 数，Adam 通常更快（EMA 加速 + 自适应学习率）。五是极端情况验证：用极小 batch（如 1）训，SGD 会剧烈震荡甚至发散（单样本梯度噪声极大），Adam 仍能收敛（EMA 抵消噪声）。证明逻辑是"同任务只改优化器，多维度对比梯度质量"，Adam 应在平滑度/一致性/收敛速度上全面占优。

### 第三层：根因深挖

**Q：偏差修正 m_hat = m/(1-β1^t) 在 t→∞ 时自动失效（β1^t→0，分母→1）。但前几步它很关键。如果不做偏差修正，前几步会发生什么？为什么不能直接初始化 m_0=g_1 跳过修正？**

不做偏差修正，前几步训练会严重"步长偏小"，训练慢甚至卡住。原因是 EMA 的初始化偏差：m_0=0，t=1 时 m_1 = β1·0 + (1-β1)·g_1 = 0.1·g_1（β1=0.9），只有真实梯度的 10%，严重低估。v_1 = (1-β2)·g_1² = 0.001·g_1²（β2=0.999），低估更严重（0.1%）。更新量 = lr·m_1/√v_1 = lr·0.1g_1/√(0.001·g_1²) = lr·0.1g_1/(0.0316·|g_1|) ≈ lr·3.16·sign(g_1)。看起来更新量反而大？问题在 v_1 的极端低估——0.001·g_1² 让 √v_1 极小，除以极小值导致更新量爆炸（方向正确但幅度失控），前几步参数剧烈跳动，可能破坏初始化（尤其 LLM 的预训练权重敏感）。偏差修正后 m_hat_1 = g_1，v_hat_1 = g_1²，更新量 = lr·g_1/|g_1| = lr·sign(g_1)，幅度受控。不能初始化 m_0=g_1 的原因：第一步前没有 g_1（m_0 是训练前初始化），且不同参数的 g_1 不同，无法统一初始化；EMA 的数学形式要求 m_0 是"无信息先验"（0 最合理）。偏差修正是"用除法补偿初始偏差"的优雅方案，t 增大自动失效，无副作用。

**Q：那如果偏差修正解决初始偏差，为什么 LLM 训练还要加 warmup？偏差修正不是已经让前几步稳定了吗？为什么不直接用峰值 lr？**

偏差修正解决了"m/v 低估"的数值问题，但没解决"初期梯度分布不稳定"的统计问题。warmup 的必要性在于：一是 v_hat 的估计初期不准——即使偏差修正让 v_hat_1 = g_1² 准确，但单步的 g_1² 不能代表该参数的长期梯度方差，v_hat 需要多步积累才反映真实分布（如某参数长期梯度大但 g_1 恰好小，v_hat 低估，后续步学习率过大）。二是 LLM 初始化敏感——随机初始化的权重在初期梯度方向混乱（各层梯度尺度差异大），大 lr 放大这个混乱，易发散；warmup 给 v_hat 积累时间（2000 步），等 v_hat 稳定后再升到峰值 lr。三是 batch 统计初期不稳——如果用 BatchNorm/LayerNorm，初期 batch 统计（均值/方差）未收敛，梯度受 batch 影响大，warmup 等统计稳定。所以偏差修正（数值层）和 warmup（统计层）解决不同问题：前者修正 m/v 的初始值偏差，后者等 v_hat 的分布稳定。两者配合才让 LLM 初期训练稳定。实证：跳过 warmup 直接峰值 lr，LLM 训练常在前几百步 loss 爆炸或 NaN；加 warmup（2000 步线性升到 3e-4）则稳定收敛。

### 第四层：方案权衡

**Q：LLM 训练常用 β2=0.95，而 CV 默认 β2=0.999。为什么 LLM 要用更小的 β2？β2 小让 v 更关注近期梯度，会不会导致自适应学习率不稳定？**

LLM 用更小 β2（0.95 vs 0.999）的原因是梯度分布的"非平稳性"。CV 任务（图像分类）的梯度分布在训练中相对稳定（特征模式固定），β2=0.999 的长窗口 EMA 准确反映长期梯度幅度。LLM 训练的梯度分布变化快：一是数据多样（不同 domain、不同长度、不同语言），不同 batch 的梯度尺度差异大；二是训练过程中模型快速变化（尤其 RLHF 阶段策略在变），梯度的统计特性非平稳；三是稀疏激活（MoE 或长尾 token），某些参数的梯度时有时无。β2=0.999 的窗口太长（约 1000 步），v 的更新滞后于实际梯度分布的变化，自适应学习率"过时"。β2=0.95（窗口约 20 步）让 v 更快反映近期梯度，自适应更敏感、更及时。会不会不稳定：β2 小确实让 v 波动更大（少平滑），但 LLM 的 lr 本身有小（3e-4）+ warmup + cosine decay 调度，且 v 的波动是"及时反映"而非"随机噪声"，整体可控。实证：GPT-3、LLaMA 等都用 β2=0.95，训练稳定；用 β2=0.999 在 LLM 上反而因 v 滞后导致后期训练效率低。这是任务特性驱动的超参选择，不是"越小越好"。

**Q：AdamW 修复了 weight decay 的耦合 bug。但为什么这个 bug 在 CV 时代不明显（Adam + L2 在 CV 用得好好的），到了 LLM 时代才必须用 AdamW？为什么不直接调大 L2 系数补偿耦合带来的衰减不足？**

bug 在 CV 不明显是因为 CV 模型的参数特性与 LLM 不同。CV 模型（CNN）的参数梯度分布相对均匀（卷积核梯度尺度相近），Adam 的"weight decay 被自适应学习率缩放"导致各参数衰减不一致，但差异不大，整体 L2 仍有效。LLM 的参数梯度分布极不均匀：embedding 层（稀疏，热门词梯度大、冷门词梯度小）、attention 的 Q/K/V（密集大梯度）、LayerNorm（小梯度）。耦合的 weight decay 让梯度大的参数（embedding 热门词）衰减被 √v 削弱（几乎不衰减），梯度小的参数（LayerNorm）衰减被放大（过度衰减）——这违背了"统一正则"的初衷，导致热门词 embedding 过拟合（不衰减）、LayerNorm 参数被过度约束（表达力下降）。调大 L2 系数补偿的问题：补偿了对梯度大的参数（热门词衰减够了），但对梯度小的参数（LayerNorm）衰减更过度，无法同时满足两类参数——耦合是结构性问题，调系数治标不治本。AdamW 的解耦让 weight decay 直接作用于参数（θ -= lr·wd·θ，不经过 √v），所有参数衰减一致，彻底解决。LLM 时代参数规模大、梯度分布极端，这个 bug 的影响被放大，必须用 AdamW。CV 时代模型小、梯度均匀，Adam+L2 凑合能用，AdamW 的增益不明显。

### 第五层：验证与沉淀

**Q：LLM 训练里 Adam 的超参（β1/β2/lr/wd）怎么选？你说 β2=0.95、lr=3e-4，但这些值怎么验证适合我的模型/数据，而不是盲目照搬？**

验证逻辑是"从经验值起步 + 小规模搜索 + 监控验证"。一是起步用经验值：β1=0.9、β2=0.95、lr=3e-4（配合 warmup）、wd=0.1，这是 GPT-3/LLaMA 的公开配置，适配大多数 LLM。二是小规模搜索（在 1-10B 小模型或数据子集上）：lr 搜索（1e-4 到 1e-3 对数扫，看哪个 loss 下降最快最稳）；β2 搜索（0.9/0.95/0.999，看训练稳定性）；wd 搜索（0.01/0.1/1，看 val loss 泛化）。三是监控信号验证：训练初期 loss 是否稳定下降（lr 合适 vs 太大发散/太小停滞）；grad_norm 是否稳定（β2 合适 vs v 估计不稳导致 grad_norm 波动）；val loss 与 train loss 的 gap（wd 合适 vs 过小过拟合/过大欠拟合）。四是扩展验证：小规模找到的超参在 full-scale 训练的前 1000 步验证，loss 稳定则确认，否则微调。五是参考文献：同规模模型的公开配置（如 LLaMA-2 7B 的 lr=3e-4、Qwen 的配置）作为基准。验证不是"调到最优"，是"确认这套超参在我的 setup 下能稳定收敛且泛化合格"，性价比最高的方式。

**Q：怎么让团队在 LLM 训练里规范地配置和调优 AdamW，而不是各自凭感觉试？**

沉淀超参配置规范和调优流程。一是配置规范：默认超参模板（β1=0.9/β2=0.95/lr=3e-4/wd=0.1/ε=1e-8 + warmup 2000 步 + cosine decay），新项目直接用模板起步。二是调优流程：先固定其他超参只调 lr（最敏感），小规模 lr sweep（1e-4/3e-4/1e-3），选 loss 下降最快的；再调 wd（看泛化），β2 一般不调（0.95 通用）。三是监控规范：训练日志必记 loss/grad_norm/lr/m_norm/v_norm/val_loss，用可视化工具（TensorBoard/W&B）实时看，异常（loss 震荡/grad_norm 爆炸/val loss 上升）立即告警。四是踩坑库：常见问题（lr 太大发散→降 lr 或加 warmup；wd 太小过拟合→升 wd；β2 太大后期低效→降到 0.95；忘记 bias correction 前几步异常→检查实现）及解决方案。五是 A/B 对比：每次超参调整跑对照实验（只改一个变量），量化效果（final val loss、训练 step 数），用数据决策。让 AdamW 调优是"模板起步 → 小规模搜索 → 监控验证 → A/B 决策"的流程，不靠"凭感觉试"。

## 结构化回答

**30 秒电梯演讲：** Adam 优化器结合动量（Momentum，一阶矩指数移动平均）和自适应学习率（RMSProp，二阶矩指数移动平均）。一阶矩 m_t=β1·m_{t-1}+(1-β1)·g_t（动量方向）。

**展开框架：**
1. **一阶矩** — 一阶矩 m_t = β1·m_{t-1}+(1-β1)·g_t（动量，梯度方向的EMA）
2. **二阶矩** — 二阶矩 v_t = β2·v_{t-1}+(1-β2)·g_t²（梯度幅度的EMA）
3. **偏差修正** — 偏差修正 m_hat=m_t/(1-β1^t), v_hat=v_t/(1-β2^t)

# 自注意力机制中为什么要除以√d_k？

> 来源：字节跳动大模型技术面试二面

## 核心问题：高维点积导致Softmax饱和

### 数学推导

假设 Query 向量 Q 和 Key 向量 K 的每个分量独立同分布，均值为0，方差为1：

```
点积: S = Q · K = Σ(i=1 to d_k) q_i · k_i

期望: E[S] = Σ E[q_i · k_i] = Σ E[q_i]·E[k_i] = 0
方差: Var[S] = Σ Var[q_i · k_i] = Σ E[q_i²]·E[k_i²] = d_k
```

**当 d_k = 128 时**（GPT-3的head维度）：
- 点积的标准差 = √128 ≈ 11.3
- 点积值范围可达 ±30 以上
- softmax(z) 其中 z 有一个值为30，其余为-30：

```
softmax([30, -30, -30, ...]) ≈ [1.0, 0.0, 0.0, ...]  ← 几乎是one-hot！
```

### 为什么one-hot有害？

```
softmax梯度 = p_i × (1 - p_i) × ∂z_i/∂θ

当 p_i ≈ 1 或 p_i ≈ 0 时：
    p_i × (1 - p_i) ≈ 0  ← 梯度消失！

┌──────────────────────────────────────────────┐
│            Softmax 梯度 vs 输入               │
│                                              │
│ 梯度  ↑     ╱╲                               │
│      │    ╱    ╲      ← 梯度敏感区           │
│      │  ╱        ╲                            │
│      │╱            ╲___                      │
│      └────┬────┬────┬────→ 输入值            │
│          -10    0   +10                      │
│       饱和区  健康   饱和区                    │
└──────────────────────────────────────────────┘
```

### 缩放的效果

```
缩放前: scores = Q·K^T        → Var = d_k = 128
缩放后: scores = Q·K^T / √d_k → Var = 1

softmax输入从 ±30 降到 ±3 左右 → 梯度健康！
```

## 代码实现

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Scaled Dot-Product Attention
    Q: (batch, heads, seq_len, d_k)
    K: (batch, heads, seq_len, d_k)
    V: (batch, heads, seq_len, d_v)
    """
    d_k = Q.size(-1)
    
    # ★ 关键：除以√d_k 进行缩放
    scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(d_k, dtype=torch.float32))
    
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    
    attn_weights = F.softmax(scores, dim=-1)
    output = torch.matmul(attn_weights, V)
    
    return output, attn_weights
```

## 为什么是√d_k而不是d_k？

**直觉**：我们需要把方差从 d_k 降到 1，标准差从 √d_k 降到 1，所以除以标准差 √d_k。

**验证**：
```
Var[S/√d_k] = Var[S] / d_k = d_k / d_k = 1  ✓
```

如果除以 d_k：`Var[S/d_k] = d_k/d_k² = 1/d_k` → 方差过小，softmax输出过于均匀（所有token权重接近），注意力退化为平均池化。

## 除了缩放还有哪些方法？

| 方法 | 原理 | 优缺点 |
|------|------|--------|
| **√d_k 缩放** | 归一化点积方差 | ✅ 标准方法，计算简单 |
| **温度参数τ** | softmax(z/τ)，τ可学习 | ✅ 更灵活；❌ 增加参数 |
| **L2归一化** | 先对Q、K做L2归一化再点积 | ✅ 稳定；❌ 改变注意力分布 |
| **相对位置编码** | 用相对位置替代绝对位置 | 缓解但不解决方差问题 |
| **RMSNorm注意力** | 用RMS替代缩放 | ✅ 计算更快；效果接近 |

## 实际影响对比

```
不缩放 (d_k=128):
  → 训练初期loss剧烈震荡
  → 梯度范数极小
  → 需要极小的学习率才能收敛
  → 最终效果显著下降

正确缩放:
  → 训练稳定
  → 可以使用较大的学习率
  → 收敛速度快10倍以上
```

**面试加分点**：提到这是Transformer原论文（Attention is All You Need, 2017）提出的标准做法；提到在低维度（d_k ≤ 64）时缩放影响较小，但在高维度（d_k ≥ 128）时至关重要；提到Flash Attention等高效实现中也保留了这一缩放。

## 记忆要点

- 核心原因防饱和：高维点积方差大(d_k)，导致Softmax输出逼近one-hot引发梯度消失。
- 数学推导记方差：Q和K点积方差为d_k，除以根号d_k刚好把方差缩放回1。
- 除以d_k不可取：若除d_k方差变1/d_k，分布过度平滑导致注意力退化为均值池化。

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：Attention 里点积除以 $\sqrt{d_k}$ 你说是"防 Softmax 饱和"。为什么饱和会导致梯度消失？数学上怎么推导？**

推导从方差开始。假设 Q 和 K 的元素是均值 0、方差 1 的独立随机变量，点积 $q \cdot k = \sum_{i=1}^{d_k} q_i k_i$ 的方差是 $d_k$（$d_k$ 个独立项求和，方差相加）。当 $d_k$ 大（如 64 或 128），点积的值域可达 $\pm\sqrt{d_k} \times 3 = \pm 24$（3 倍标准差），Softmax 输入差距巨大，输出逼近 one-hot（最大值接近 1，其余接近 0）。Softmax 的梯度是 $\text{softmax}_i \times (1 - \text{softmax}_i)$，当输出接近 one-hot 时，最大值的梯度 $\approx 1 \times (1-1) = 0$，其余的梯度 $\approx 0 \times (1-0) = 0$，梯度几乎为零，反向传播无法更新权重。除以 $\sqrt{d_k}$ 把方差缩放回 1，点积值域回到 $\pm 3$，Softmax 输出平滑（如 0.3/0.3/0.4），梯度正常。

### 第二层：证据与定位

**Q：训练 Transformer 时 loss 早期降不下来（卡在高值）。你怎么判断是 attention 的 $\sqrt{d_k}$ 缩放问题、还是学习率/初始化问题？**

看 attention 权重分布和梯度统计。一是 attention 权重——打印每层的 attention matrix（softmax 后的权重），如果接近 one-hot（某一行只有一个 1，其余全 0），是 Softmax 饱和（$\sqrt{d_k}$ 缩放没生效或 $d_k$ 太大），信息只关注一个 token，梯度消失；如果 attention 权重均匀（每行都是 1/N），则是没学到有意义的 attention（可能是初始化差或学习率太小）。二是梯度统计——各层的梯度范数（gradient norm），如果某些层梯度接近 0（如 <1e-7），是梯度消失（可能是 Softmax 饱和导致，也可能是深层网络梯度链式衰减）；如果梯度爆炸（>1e3），是学习率太大。三是对照实验——临时去掉 $\sqrt{d_k}$ 缩放（直接用 $qk$），如果 loss 更差，说明缩放是必要的；如果去掉反而好（罕见），可能是初始化已适应无缩放。

### 第三层：根因深挖

**Q：除以 $\sqrt{d_k}$ 把方差缩放到 1。但如果 $d_k$ 很小（如 8），不缩放也没事，为什么小模型也保留缩放？**

一致性和稳定性。小 $d_k$（如 8）时点积方差 8，值域 $\pm 8.5$，Softmax 输入差距虽小于大 $d_k$ 但仍有饱和风险（输出可能 0.9/0.05/0.05，梯度偏小）。更重要的是"架构一致性"——Transformer 从小到大（7B 到 70B），$d_k$ 从 64 到 128 变化，统一用 $\sqrt{d_k}$ 缩放保证不同规模的训练行为一致（不缩放的模型在小规模能训但大规模崩，实验不可复现）。且 $\sqrt{d_k}$ 缩放是无害的——即使 $d_k$ 小（不需要缩放），缩放后只是把 Softmax 输入整体缩小，不影响 attention 的相对顺序（最大值仍最大），只是让输出更平滑。所以保留缩放是"零风险 + 一致性"的选择。

**Q：那为什么不除以 $d_k$（而非 $\sqrt{d_k}$），把方差压得更平，岂不是更不容易饱和？**

除以 $d_k$ 会过度平滑，导致 attention 退化。除以 $d_k$ 后方差变 $1/d_k$（如 $d_k=64$ 时方差 1/64），点积值域 $\pm 0.375$，Softmax 输入几乎相等，输出接近均匀分布（1/N），attention 退化成"均值池化"（所有 token 等权平均），失去了"选择性关注"的能力。Attention 的核心价值是"聚焦重要 token"（输出接近 one-hot），缩放的目的是"适度聚焦"（输出如 0.5/0.3/0.2，既聚焦又不饱和），$\sqrt{d_k}$ 正好平衡——方差 1 让 Softmax 输入差距适中（既不平到均匀，也不尖到 one-hot）。$\sqrt{d_k}$ 是信息论意义上的"最优缩放"（让点积的方差与 Q/K 的方差一致），数学上最自然。

### 第四层：方案权衡

**Q：现代大模型（如 Llama-2）的 $d_k$ 是 128。为什么选 128 而非更大（如 256）或更小（如 64）？**

$d_k$ 影响"每个头的表达能力"和"计算量"。$d_k$ 大，每个头能捕捉更丰富的 attention 模式（Q/K 向量维度高，表达力强），但计算量大（点积是 $O(d_k)$）且容易饱和（方差大）；$d_k$ 小，计算快但表达力弱。Llama-2 选 $d_k=128$ 是经验值——在 7B/13B/70B 上都表现良好，128 维的 Q/K 足够捕捉常见 attention 模式，且配合 $\sqrt{128}\approx 11.3$ 缩放，Softmax 不会饱和。多头机制（Multi-Head）弥补单头维度的限制——多个 $d_k=128$ 的头并行，各头关注不同模式（有的关注语法、有的关注语义），综合后表达力足够。$d_k$ 的选择和 head 数（$h$）、hidden_dim（$d_{model}$）耦合：$d_{model} = h \times d_k$，Llama-2-7B 是 $h=32, d_k=128, d_{model}=4096$。

**Q：为什么不用 ReLU 或其他激活替代 Softmax 做 attention（如 ReLU 后归一化），省得担心 Softmax 饱和？**

替代激活会破坏 attention 的"概率解释"。Softmax 的输出是概率分布（非负、和为 1），attention 权重可解释为"关注各 token 的概率"，V 的加权求和是"期望值"。ReLU 后归一化（如 ReLU(qk) / sum(ReLU(qk))）虽然避免指数运算（Softmax 的 exp 是数值不稳定根源），但 ReLU 会把负的点积截断为 0（丢失"负相关"信息），且归一化后不保证所有权重为正（如果有负值）。Softmax 的指数运算天然保证非负 + 归一化，数学性质好。且现代硬件（GPU）对 Softmax 优化好（Flash Attention 把 Softmax 融合进 kernel，避免中间结果写回显存），Softmax 的计算开销已不是瓶颈。所以 Softmax 仍是主流，替代方案（如 ReLU attention、Linear Attention）在研究中但未主流。

### 第五层：验证与沉淀

**Q：你怎么证明 $\sqrt{d_k}$ 缩放是必要的，而非可有可无？**

做消融实验。在相同模型（如 6 层 Transformer，$d_k=64$）和相同数据上，训三组：无缩放（直接 $qk$）、$\sqrt{d_k}$ 缩放、$d_k$ 缩放。对比：一是训练 loss 曲线——无缩放的 loss 早期卡住（Softmax 饱和、梯度消失），$\sqrt{d_k}$ 正常收敛，$d_k$ 收敛但最终 loss 略高（过度平滑）。二是 attention 权重分布——无缩放的 attention 接近 one-hot，$\sqrt{d_k}$ 适中，$d_k$ 接近均匀。三是下游任务表现——$\sqrt{d_k}$ 最优。这是经典论文（"Attention Is All You Need"）验证过的结论，$d_k$ 越大差异越明显（$d_k=64$ 时无缩放还能训，$d_k=512$ 时完全训不动）。验证方法：可视化 attention 权重热力图 + 统计 attention 熵（熵低=集中=饱和，熵高=分散=均匀）。

**Q：Attention 的 $\sqrt{d_k}$ 缩放怎么沉淀成模型设计的常识？**

固化成"Transformer 架构检查清单"：默认所有 attention 计算必须除以 $\sqrt{d_k}$（或等价的缩放），$d_k$ 选 64 或 128（配合 head 数），初始化时 Q/K 的方差控制在 1（配合缩放让点积方差稳定）。沉淀"$d_k$ 与模型规模的对照表"（小模型 $d_k=64$，大模型 $d_k=128$）、"attention 健康度监控"（训练时打印 attention 熵，过低=饱和、过高=退化）。把"$\sqrt{d_k}$ 缩放"作为 Transformer 实现的必选项（不可省略），代码 review 时检查 attention 实现是否含缩放（某些 bug 是漏了缩放）。配套训练监控（attention 熵、梯度范数），发现饱和及时排查。

## 结构化回答

**30 秒电梯演讲：** 除以√d_k是为了控制点积结果的方差，防止softmax落入梯度消失的饱和区——就像调音量的旋钮。

**展开框架：**
1. **点积方差** — 点积方差 = d_k × σ²，维度越高方差越大
2. **softmax** — softmax在大输入时梯度趋近零（饱和区），导致训练停滞
3. **除以√d_k** — 除以√d_k将方差归一化为1，保持梯度健康

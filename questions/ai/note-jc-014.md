# 【阶跃星辰面经】动态规划核心思想 + 最短路逆推子问题

## 一、动态规划核心思想

**核心**：把大问题拆成子问题，**存储子问题的解**，避免重复计算。

**适用条件**（必须同时满足）：
1. **最优子结构**：大问题的最优解包含子问题的最优解
2. **重叠子问题**：子问题被反复计算（暴力递归会指数爆炸）

## 二、DP 的两大要素

```
[1] 状态定义：dp[...] 代表什么
    例：dp[i] = 以第 i 个元素结尾的最长子序列长度
    
[2] 状态转移方程：dp[i] 怎么从子问题算出来
    例：dp[i] = max(dp[j] + 1) for all j < i 且 arr[j] < arr[i]
```

## 三、经典例子：斐波那契（重叠子问题）

```python
# 暴力递归：O(2^n)（重复计算 fib(3) 无数次）
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)

# DP（记忆化）：O(n)（每个 fib(i) 只算一次）
memo = {}
def fib(n):
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib(n-1) + fib(n-2)
    return memo[n]

# DP（迭代）：O(n)
def fib(n):
    if n <= 1: return n
    dp = [0]*(n+1)
    dp[1] = 1
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

**为什么 DP 快**：暴力递归 fib(5) 会算 fib(3) 两次、fib(2) 三次...指数级重复。DP 存起来只用算一次。

## 四、经典例子：0-1 背包（最优子结构）

```
问题：n 个物品（重量w[i]、价值v[i]），背包容量 W，求最大价值。

状态定义：dp[i][j] = 前 i 个物品、容量 j 时的最大价值
状态转移：
  不选第 i 个：dp[i][j] = dp[i-1][j]
  选第 i 个：  dp[i][j] = dp[i-1][j-w[i]] + v[i]  （j >= w[i]）
  取最大：dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

最优子结构：dp[i][j] 的最优来自 dp[i-1][...] 的最优。
```

## 五、最短路问题的逆推解法

### 问题定义
给定有向图，求从起点 s 到终点 t 的最短路径。

### 逆推解法（从终点倒推）

**子问题定义**：
```
dp[i] = 从节点 i 到终点 t 的最短距离

边界：dp[t] = 0（终点到终点距离 0）
目标：dp[s]（起点到终点最短距离）

状态转移（逆推）：
  dp[i] = min over all edges (i→j):  weight(i,j) + dp[j]
  
  即：从 i 出发，看所有相邻节点 j，
      "走 i→j 这条边 + 从 j 到终点的最短距离"，
      取最小。
```

### 示例

```
图：
  s --3--> A --2--> t
  s --5--> B --1--> t
  A --4--> B

逆推：
  dp[t] = 0
  dp[B] = w(B,t) + dp[t] = 1 + 0 = 1
  dp[A] = min(w(A,t)+dp[t], w(A,B)+dp[B])
        = min(2+0, 4+1) = min(2, 5) = 2
  dp[s] = min(w(s,A)+dp[A], w(s,B)+dp[B])
        = min(3+2, 5+1) = min(5, 6) = 5

最短路径：s→A→t，距离 5
```

### 子问题是什么

```
子问题 = "从某个节点到终点的最短距离"
  dp[A] 的子问题是 dp[t] 和 dp[B]（A 的邻居）
  dp[s] 的子问题是 dp[A] 和 dp[B]（s 的邻居）

最优子结构：dp[s] 的最优 = min(边(s,邻居) + 邻居的最优 dp[邻居])
重叠子问题：如果有多条路径经过同一节点，该节点的 dp 值被多次复用
```

## 六、逆推 vs 顺推（Dijkstra）

```
逆推（DP）：从终点倒推，适合 DAG（无环图）
  dp[i] = min(w(i,j) + dp[j])
  
顺推（Dijkstra）：从起点正向扩展，适合带非负权的任意图
  用优先队列每次取最近节点扩展

DAG 最短路用逆推 DP（拓扑序处理）更简单；
有环图用 Dijkstra（DP 会循环依赖）。
```

## 七、加分点

- 说出 **DP 的两个适用条件**：最优子结构 + 重叠子问题
- 说出 **状态定义和状态转移方程是 DP 的核心**
- 说出 **最短路逆推的子问题是"从某节点到终点的最短距离"**

## 八、雷区

- ❌ 混淆贪心和 DP：贪心不回退（局部最优），DP 考虑所有子问题（全局最优）
- ❌ 状态定义错误：dp[i] 含义不清导致转移方程写错
- ❌ 忽略边界条件：dp[0] / dp[t] 等初始值要正确

## 九、扩展

- **自顶向下（记忆化递归）vs 自底向上（迭代）**：两者等价，迭代通常更快（无递归开销），记忆化更直观
- **空间优化**：背包问题 dp[i][j] 只依赖 dp[i-1][...]，可压缩成一维 dp[j]
- **区间 DP / 树形 DP / 状压 DP**：按状态结构分类的高级 DP 技巧

## 记忆要点

- 核心思想：大问题拆子问题并存储解，空间换时间避免重复计算
- 两大要素：状态定义 dp[i] 与状态转移方程 dp[i] 的推演公式
- 适用条件：必须具备最优子结构与重叠子问题两大特征
- 最短路逆推：定义 dp[i] 为到终点最短距离，公式 min(w(i,j)+dp[j])
- 对比差异：逆推 DP 适合无环图(DAG)，有环图必须用 Dijkstra

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：DP 的核心是"存子问题解避免重复计算"。但有些问题子问题不重叠（如分治的归并排序，子数组不重复）。DP 和分治的区别到底是什么？怎么判断一个问题该用 DP 还是分治？**

DP 和分治都"拆大问题为子问题"，区别在于"子问题是否重叠"。分治（如归并排序）：子问题互不重叠（归并排序的左半和右半是不同的元素，各自独立排序，无共享子问题），每个子问题只解一次，无需存储——递归自然解决，不存中间结果。DP（如斐波那契、背包）：子问题重叠（fib(5) 依赖 fib(4) 和 fib(3)，fib(4) 又依赖 fib(3) 和 fib(2)，fib(3) 被多次计算），暴力递归会指数爆炸，DP 通过存储（记忆化或迭代）让每个子问题只解一次。判断方法：分析问题的递归结构，如果子问题有共享（同一子问题被多个父问题依赖），用 DP；如果子问题独立（每个只被一个父问题依赖一次），用分治。补充条件：DP 还需"最优子结构"（大问题最优解含子问题最优解），这是优化问题（求最值）的特征；分治不要求最优子结构（任何可分解的问题都能分治，如排序、FFT）。实操判断：一是画递归树，看是否有重复节点（重叠）——有则 DP，无则分治；二是问题是否求最值（最大/最小/最长）且子问题的选择影响后续——是则 DP；三是暴力递归是否超时（指数级）——是则需 DP 存储优化。最短路（子问题"某节点到终点最短距离"被多条路径共享）用 DP；排序（子问题"左半排序""右半排序"独立）用分治。

### 第二层：证据与定位

**Q：你说 DP 适用需"最优子结构"和"重叠子问题"。但实际拿到一个问题，怎么快速判断它有这两个特征？有没有具体的检验方法，而不是凭感觉？**

有具体检验方法。最优子结构的检验：假设大问题的最优解，看它"是否由子问题的最优解构成"。具体步骤：一是写出大问题的决策（如最短路的"下一步走哪个邻居"）；二是假设大问题最优解做了某个决策（选了邻居 j），那么剩余子问题"从 j 到终点的最优"必须也是最优的（否则可以用更优的替换，矛盾）——如果这个"替换论证"成立，有最优子结构。反例：最长简单路径（不能重复访问节点）没有最优子结构——s→t 的最长路径选了邻居 j 后，"j→t 的最长路径"可能访问了 s 已访问的节点，与"简单路径"约束冲突，子问题不能独立优化。重叠子问题的检验：画递归树（或分析状态空间），看子问题的数量是否远小于递归调用的次数。具体：一是定义状态（如 dp[i]），看状态的总数（如 n 个 i，共 n 个状态）；二是看暴力递归的调用次数（如 fib 的递归树有 2^n 节点）；三是如果调用次数 >> 状态数（如 2^n >> n），说明大量重复（重叠），需 DP 存储。反例：归并排序的子问题是 (arr, low, mid) 和 (arr, mid+1, high)，每个子问题唯一（不同的数组段），不重叠，无需 DP。实操：先尝试暴力递归，分析时间复杂度——如果指数级（如 O(2^n)），大概率有重叠子问题，改 DP；如果多项式（如 O(n log n)），可能不重叠，分治即可。

### 第三层：根因深挖

**Q：最短路逆推 DP 要求图是 DAG（无环）。但带权图常有环。为什么有环不能用逆推 DP？DP 的状态转移 dp[i] = min(w(i,j) + dp[j]) 在有环图里会发生什么？**

有环图的逆推 DP 会陷入"循环依赖"，无法收敛。原因是 DP 的前提是"子问题有明确的计算顺序"（拓扑序），即计算 dp[i] 时，它依赖的所有 dp[j] 必须已计算完。DAG（无环图）有拓扑序（如逆拓扑序：先算终点 dp[t]，再算其前驱，...），按拓扑序计算每个 dp[i] 时 dp[j] 已就绪。有环图的环（如 A→B→A）破坏了拓扑序——计算 dp[A] 需要 dp[B]（A→B），但 dp[B] 又需要 dp[A]（B→A），互相依赖，无法确定计算顺序。具体表现：如果强行迭代计算，dp[A] 和 dp[B] 会互相更新，可能不收敛（值在循环）或收敛到错误值（负权环可无限减小）。更严重的是负权环（环的总权为负），最短路径无意义（绕环无限次总长趋 -∞），DP 无法处理。Dijkstra 算法处理有环图的方式：不用 DP 的"子问题顺序"，而是用贪心 + 优先队列——从起点扩展，每次取当前最近节点（已确定最短路径），松弛其邻居。贪心的正确性依赖"非负权"（已确定节点的最短路径不会被后续更新，保证贪心选择正确），且天然处理环（已访问节点不重复扩展）。所以有环图（非负权）用 Dijkstra，DAG 用逆推 DP（拓扑序更简单高效）。如果图有负权边（无负权环），用 Bellman-Ford（支持负权，能检测负权环）；有负权环则最短路无解。

**Q：那如果问题是 DAG 最短路，逆推 DP 和 Dijkstra 都能用。选哪个？两者的复杂度和实现差异是什么？为什么不统一用 Dijkstra（看起来更通用）？**

DAG 最短路优先用逆推 DP，因为更简单高效。复杂度对比：逆推 DP 是 O(V+E)（V 节点，E 边，每个节点算一次 dp[i]，每次遍历其出边松弛邻居，总 O(V+E)）；Dijkstra 用优先队列是 O((V+E) log V)（每条边松弛一次，每次松弛可能更新优先队列，优先队列操作 O(log V)）。逆推 DP 更快（O(V+E) < O((V+E) log V)）。实现差异：逆推 DP 需先做拓扑排序（DAG 可拓扑排序，O(V+E)），然后按逆拓扑序计算 dp（确保 dp[j] 在 dp[i] 之前计算），实现简单（拓扑排序 + 一次遍历）。Dijkstra 需优先队列（堆），实现稍复杂（维护 dist 数组、堆操作）。Dijkstra 在 DAG 上"浪费"了通用性——它为有环图设计，DAG 上没有环不需要它的贪心+堆机制，逆推 DP 直接利用 DAG 的拓扑序更高效。统一用 Dijkstra 的代价：一是复杂度多一个 log V（堆操作）；二是实现复杂（优先队列）；三是不能处理负权（Dijkstra 要求非负权，逆推 DP 在 DAG 上可处理负权，只要无负权环）。所以 DAG 最短路用逆推 DP（简单、快、支持负权），有环图用 Dijkstra（通用）。这也是为什么分析问题时要先判断图结构（DAG vs 一般图）——DAG 有更优算法（拓扑排序 + DP），一般图只能用 Dijkstra/Bellman-Ford。最优化是"按问题结构选算法"，不是"用最通用的"。

### 第四层：方案权衡

**Q：DP 有自顶向下（记忆化递归）和自底向上（迭代）两种实现。两者等价，但工程上选哪个？它们在性能、可读性、调试上各有什么优劣？**

两种实现的权衡。自顶向下（记忆化递归）：从大问题出发，递归调子问题，用 memo 字典存已算结果，首次计算时存，后续直接读。优势：一是直观（直接翻译递推关系，代码结构反映问题结构）；二是懒计算（只算实际需要的子问题，如果某些状态不可达则不计算）；三是思考成本低（定义递归函数即可）。劣势：一是递归开销（函数调用、栈空间，深度大时栈溢出）；二是 memo 查询开销（哈希表查询，常数大）；三是缓存不友好（递归的内存访问模式随机）。自底向上（迭代）：从最小子问题出发，按顺序填充 dp 表，直到大问题。优势：一是性能好（无递归开销，数组顺序访问缓存友好，常数小，通常快 2-5 倍）；二是空间可优化（如背包的 dp[i][j] 只依赖 dp[i-1][j]，可压缩成一维 dp[j]，省空间）；三是无栈溢出风险。劣势：一是思考顺序反（需确定计算顺序，如背包按 i 从小到大、j 从大到小）；二是计算所有状态（即使某些不需要，浪费）；三是边界处理繁琐（dp[0][0] 等初始值要正确）。工程选择：一是性能敏感（竞赛、生产）用自底向上（快、省空间）；二是问题复杂、递推关系不直观时先用自顶向下理清思路（可读性优），验证正确后转自底向上优化；三是状态空间稀疏（多数状态不可达）用自顶向下（只算需要的）。实践中，简单 DP（斐波那契、背包）用自底向上，复杂 DP（树形、状态压缩）先自顶向下理清再转换。

**Q：0-1 背包和完全背包的 DP 转移方程相似，但实现细节不同（0-1 背包 j 逆序，完全背包 j 顺序）。为什么循环顺序影响结果？一维空间优化下顺序错了会怎样？**

循环顺序影响"每个物品是否可重复使用"。0-1 背包（每物品最多用一次）：二维 dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])，依赖上一行 i-1。一维优化（压缩掉 i 维）时，dp[j] = max(dp[j], dp[j-w[i]] + v[i])，如果 j 正序（从小到大），计算 dp[j] 时 dp[j-w[i]] 可能已被当前物品 i 更新（因为 j-w[i] < j，先算），导致物品 i 被"重复使用"（用了多次，违反 0-1 约束）。j 逆序（从大到小）时，计算 dp[j] 时 dp[j-w[i]] 还是上一轮（物品 i-1）的值（因为 j-w[i] < j，后算，未被 i 更新），保证物品 i 只用一次。完全背包（每物品可用无限次）：正好需要"物品可重复使用"，所以 j 正序——计算 dp[j] 时允许用当前物品 i 已更新的 dp[j-w[i]]（相当于物品 i 用了多次）。一维优化下顺序错了的后果：0-1 背包用正序（应逆序），物品被重复使用，结果错误（偏大，如价值虚高）；完全背包用逆序（应正序），物品不能重复使用，退化成 0-1 背包（结果偏小，非最优）。所以循环顺序是实现"物品使用次数约束"的关键，一维优化必须按约束选顺序。理解原理：一维 dp 是"滚动数组"，j 的顺序决定"读到的 dp[j-w[i]] 是上一轮（物品 i-1）还是本轮（物品 i）的值"，进而决定物品 i 是否可重复。二维 dp 无此问题（显式区分 i-1 和 i 行），但空间大。一维优化的顺序是这个细节的"隐式编码"，容易出错，需理解原理而非死记。

### 第五层：验证与沉淀

**Q：DP 解法的正确性怎么验证？状态转移方程写对了，但怎么确认"没漏状态""边界正确""最终结果对"？**

多层验证。一是小规模暴力对照：对小 n（如 n=5-10），用暴力枚举（穷举所有可能）算正确答案，对比 DP 结果，一致则 DP 大概率对。这是最可靠的验证（暴力是 ground truth）。二是边界检查：dp[0]、dp[1] 等初始值是否正确（如斐波那契 dp[0]=0, dp[1]=1；背包 dp[0][j]=0 容量为 0 时价值 0）；递归终止条件是否覆盖（如 n≤1 时直接返回）；状态空间的边界（如 j-w[i] ≥ 0 才能选物品 i）。漏边界是常见 bug。三是状态完备性：分析问题时确认状态定义是否完备（如背包的 dp[i][j] 是否覆盖所有"前 i 物品、容量 j"的情况；最长上升子序列的 dp[i] 是否覆盖所有"以 i 结尾"的情况）。漏状态（如遗漏某类情况）导致结果错。四是转移正确性：dp[i] 的转移是否覆盖所有决策（如背包的"选"和"不选"两种；最短路的"走每个邻居"）。五是测试用例：构造多种测试用例（小规模、大规模、边界值、特殊输入如全相同、已序），与已知答案（如题目样例、暴力对照）对比。六是逻辑推理：对状态转移方程做"替换论证"（假设子问题最优，证明大问题最优），验证最优子结构。证明逻辑是"小规模暴力对照 + 边界检查 + 状态完备 + 转移正确 + 多测试 + 逻辑推理"，多层验证确保 DP 正确。如果某层失败（如暴力对照不符），定位是状态定义、转移方程还是边界问题，迭代修正。

**Q：怎么让团队在面对新问题时，系统地"判断是否用 DP + 定义状态 + 写转移方程"，而不是靠灵感或套模板？**

沉淀 DP 解题方法论和训练流程。一是判断流程：问题求最值（最大/最小/最长/最短）→ 尝试暴力递归 → 分析是否有最优子结构（替换论证）和重叠子问题（递归树重复）→ 确认则用 DP。二是状态定义方法：状态 = "问题的参数"（如背包的"前 i 物品、容量 j"；序列的"以 i 结尾"；区间的"[i,j]"）；从"暴力递归的参数"提炼状态（递归函数的参数就是状态）。三是转移方程方法：枚举"当前步骤的所有决策"（背包的选/不选；最短路的走哪个邻居），每个决策对应一个子问题，取最优（max/min）。四是模板库：按问题类型（序列 DP、背包、区间、树形、状压、数位）整理模板和经典例题，新人按类型学。五是训练流程：从经典问题（斐波那契、背包、LCS、编辑距离）起步，掌握状态定义和转移；进阶到变体（0-1/完全/多重背包、区间 DP、树形 DP）；最后挑战综合题（状压、数位）。六是踩坑库：常见错误（状态定义不清、转移遗漏决策、边界初始值错、一维优化顺序错）及案例。让 DP 解题是"判断流程 → 状态定义 → 转移推导 → 验证"的方法论，配合模板库和训练，从"靠灵感"到"系统化"。

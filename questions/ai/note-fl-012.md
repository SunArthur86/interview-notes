# 【字节飞连面经】缓存穿透 / 击穿 / 雪崩：含义和解法，布隆过滤器

## 一、三大问题对比

| 概念 | 含义 | 解法 |
|------|------|------|
| **穿透** | 查一个**不存在**的 key，每次都打到 DB | 缓存空值（短 TTL）+ **布隆过滤器**前置判断 |
| **击穿** | 一个**热点 key 过期**瞬间大量并发打到 DB | **互斥锁**（只让一个线程回源）/ 永不过期 + 异步更新 |
| **雪崩** | **大量 key 同时过期** + 突发流量打到 DB | **过期时间加随机扰动** / 多级缓存 / 限流降级 |

## 二、穿透：缓存空值 + 布隆过滤器

```
查询 user:999（不存在）
  │
  ├─[无布隆过滤器] Redis miss → DB miss → 返回 null
  │                                  ↑ 每次都打 DB（被恶意攻击就挂）
  │
  └─[有布隆过滤器] 布隆过滤器先判断 → 说不存在 → 直接返回（不查 DB）
                                         说存在 → 可能误判 → 查 Redis/DB
```

**缓存空值**：DB miss 后把 `user:999 → null` 缓存（短 TTL，如 60s），下次直接命中。

**布隆过滤器**：
- 用多个 hash 函数把 key 映射到 bit 数组
- **说不存在一定不存在**（100% 准确）
- **说存在可能误判**（false positive）
- 用来挡掉绝大部分"查不存在 key"的穿透

## 三、击穿：互斥锁防并发回源

```
热点 key 过期瞬间，1000 个请求同时来
  │
  ├─[无锁] 1000 个请求都 Redis miss → 1000 个都打 DB → DB 挂
  │
  └─[互斥锁] 
       第 1 个请求拿到锁 → 回源 DB → 写回 Redis → 释放锁
       其余 999 个请求等锁 → 锁释放后读 Redis（命中）
```

**互斥锁实现**：`SETNX lock:key 1 EX 10`（拿锁，10s 过期防死锁）。

**另一解**：永不过期 + 异步更新（后台定时刷新，热点 key 永远在缓存）。

## 四、雪崩：过期时间加随机扰动

```
[问题] 大量 key 同时过期（如批量预热的缓存同一时刻 TTL 到）
       → 突发流量全部打 DB → DB 雪崩

[解法] 过期时间 = base + random(0, jitter)
       TTL = 3600 + rand(0, 600)   # 1 小时 ± 10 分钟随机
       → 打散过期时间点
```

**其他解法**：
- 多级缓存（本地缓存 Caffeine + Redis + DB）
- 限流降级（Hystrix / Sentinel 保护 DB）

## 五、布隆过滤器详解

| 特性 | 说明 |
|------|------|
| 空间 | 极省（1 亿 key 约 100MB） |
| 查询 | O(k)，k=hash 函数个数 |
| **说"不存在"** | **100% 准确** |
| **说"存在"** | **可能误判**（false positive） |
| 删除 | 标准布隆过滤器**不能删**（bit 不能复位） |

**误判率控制**：增加 bit 数组大小 + 增加 hash 函数个数。典型配置 1 亿 key 误判率 1% 约 114MB。

**Counting Bloom Filter**：用计数器代替 bit，支持删除（但空间翻几倍）。

## 六、加分点

- 说出 **布隆过滤器不能删**，要删得用 Counting Bloom Filter 或 Cuckoo Filter
- 说出 **互斥锁要设过期时间**（防持锁线程崩溃死锁）
- 说出 **雪崩 vs 击穿的区别**：击穿是单 key，雪崩是多 key

## 七、扩展

- **缓存预热**：服务启动时主动加载热点 key，避免冷启动雪崩
- **多级缓存一致性**：本地缓存（Caffeine）+ 分布式缓存（Redis），更新时本地缓存怎么失效（广播失效 / TTL 短）
- **Redis 集群的雪崩**：不仅是 key 过期，主节点宕机切换期间也会"雪崩"——需要熔断降级保护

## 记忆要点

- 穿透(查不存在)解法：缓存空值短TTL，配合布隆过滤器前置拦截挡掉恶意请求
- 击穿(热Key过期)解法：互斥锁(SETNX)只让单线程回源查DB，或永不过期加异步更新
- 雪崩(大量同时过期)解法：过期时间加随机扰动打散，配合多级缓存与限流降级
- 布隆过滤器特性：说不存在100%准，说存在有误判，且标准版不支持删除元素

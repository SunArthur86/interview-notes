# 【月之暗面面经】如何设计 AI 桌面端的产物 diff，既能给新手看，也能给高手看？

## 一、三级Diff分层模型

```
┌──────────────────────────────────────────────────────────────────┐
│                    Diff 信息分层金字塔                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│           Level 3: 原始Diff（给开发者/极客）                      │
│          ┌───────────────────────┐                               │
│          │ 字符级/行级原始对比     │                               │
│         └───────────────────────┘                                │
│       Level 2: 结构化Diff（给有经验用户）                          │
│      ┌───────────────────────────────┐                           │
│      │ 按区块/段落/页面的语义对比      │                           │
│     └───────────────────────────────┘                            │
│   Level 1: 摘要Diff（给所有用户/新手）                             │
│  ┌───────────────────────────────────┐                           │
│  │ "改了3处：标题更新+数据修正+新增表格" │                           │
│  └───────────────────────────────────┘                           │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 五、大产物的页面级Diff

对于站点和PPT等大产物，diff不能按行展示，需要按页面/幻灯片展示：

```
┌──────────────────────────────────────────────────────────────────┐
│  📑 PPT Diff：分析报告.pptx                        幻灯片对比      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  第1页  无变化 ✓                                                │
│  第2页  无变化 ✓                                                │
│  第3页  ⚠️ 有修改                                    [对比 ▼]   │
│  ┌──────────────────────────┬──────────────────────────┐        │
│  │      原始 (v1)            │      修改后 (v2)          │        │
│  │                          │                          │        │
│  │   市场增长 30%            │   市场增长 45%            │        │
│  │   (2024 Q3数据)           │   (2024 Q4数据)          │        │
│  │                          │                          │        │
│  │   📊 趋势图(旧)           │   📊 趋势图(新)           │        │
│  └──────────────────────────┴──────────────────────────┘        │
│  变更摘要：更新数据源Q3→Q4，图表数据随之更新                      │
│  [ 接受此页修改 ]  [ 拒绝 ]  [ 局部编辑 ]  [ 重新生成此页 ]     │
│                                                                  │
│  第4页  新增 ➕                                     [查看 ▼]    │
│  第5页  无变化 ✓                                                │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 六、实现：Diff渲染器

```typescript
class DiffRenderer {
  render(artifact: Artifact, diff: ArtifactDiff, level: DiffLevel) {
    switch (level) {
      case 'summary':
        return this.renderSummary(diff);
      case 'structured':
        return this.renderStructured(artifact, diff);
      case 'raw':
        return this.renderRaw(diff);
    }
  }
  
  // 摘要diff
  private renderSummary(diff: ArtifactDiff): SummaryView {
    return {
      totalChanges: diff.changes.length,
      categories: this.categorizeChanges(diff.changes),
      // { modified: 2, added: 1, deleted: 0 }
      changes: diff.changes.map(c => ({
        type: c.type,
        description: this.humanizeChange(c),
        impact: c.impact,
      })),
    };
  }
  
  // 根据用户专业度自动选择默认diff层级
  recommendLevel(userProfile: UserProfile): DiffLevel {
    if (userProfile.totalTasks < 10) return 'summary';
    if (userProfile.totalTasks < 50) return 'structured';
    return 'raw';
  }
}
```

## 七、常见坑

- **只有代码级diff**：普通用户看不懂 `@@ -1,5 +1,12 @@`，直接拒绝所有修改
- **不支持局部接受**：全盘接受或全盘拒绝，用户无法精细控制
- **大产物用文本diff**：站点/PPT这种结构化产物用文本diff，信息量太大
- **没有diff层级切换**：不能根据需要切换详细程度，要么太简略要么太详细

# 向量间相似度如何计算？

## 一、三种常用相似度

```
┌──────────────────────────────────────────────────┐
│              三种向量相似度度量                      │
├──────────────────────────────────────────────────┤
│                                                    │
│  1. 余弦相似度（Cosine Similarity）— 最常用        │
│     衡量"方向"的接近程度                           │
│     范围：[-1, 1]，越大越相似                      │
│                                                    │
│  2. 点积（Dot Product）                            │
│     方向 × 模长                                    │
│     受向量长度影响                                 │
│                                                    │
│  3. 欧氏距离（Euclidean Distance）                 │
│     绝对距离                                       │
│     范围：[0, +∞)，越小越相似                      │
│                                                    │
└──────────────────────────────────────────────────┘
```

## 二、余弦相似度（最常用）

```python
import numpy as np

def cosine_similarity(vec_a, vec_b):
    """余弦相似度：衡量方向夹角"""
    dot = np.dot(vec_a, vec_b)              # 点积
    norm_a = np.linalg.norm(vec_a)          # 向量A的模长
    norm_b = np.linalg.norm(vec_b)          # 向量B的模长
    return dot / (norm_a * norm_b)

# 例：
# A = "我喜欢猫" → [0.3, 0.5, 0.1]
# B = "我爱小猫" → [0.4, 0.6, 0.2]
# cos(A,B) ≈ 0.98 （方向几乎一致，语义相似）

# C = "今天天气好" → [0.1, 0.0, 0.9]
# cos(A,C) ≈ 0.15 （方向差异大，语义不相似）

# 特点：
# - 只看方向，不看长度（模长归一化后算）
# - 适合语义相似度（语义相近=方向相近）
# - 不受文本长度影响
```

## 三、点积（Dot Product）

```python
def dot_product(vec_a, vec_b):
    """点积：方向 × 模长"""
    return np.dot(vec_a, vec_b)

# 与余弦的关系：
# dot(A,B) = |A| × |B| × cos(A,B)
# 
# 如果向量已归一化（|A|=|B|=1）：
# dot(A,B) = cos(A,B)  ← 两者相同

# 点积受模长影响：
# 长文本的Embedding模长可能更大
# 点积会偏好长文档（不公平）
# 所以RAG通常用余弦（先归一化）
```

## 四、欧氏距离

```python
def euclidean_distance(vec_a, vec_b):
    """欧氏距离：绝对距离"""
    return np.sqrt(np.sum((vec_a - vec_b) ** 2))

# 范围：[0, +∞)，越小越相似
# 特点：
# - 衡量绝对位置差异
# - 受向量模长和方向双重影响
# - 适合需要考虑"绝对差异"的场景
```

## 五、三种度量的对比

```
┌──────────────┬──────────────────┬──────────────────────┐
│ 度量          │ 关注点              │ RAG适用性              │
├──────────────┼──────────────────┼──────────────────────┤
│ 余弦相似度    │ 方向（角度）        │ ★★★ 最常用            │
│              │ 不受模长影响        │ 不受文本长度影响        │
├──────────────┼──────────────────┼──────────────────────┤
│ 点积          │ 方向 × 模长        │ ★★ 向量归一化后=余弦  │
│              │ 受模长影响          │ 计算更快(不用归一化)   │
├──────────────┼──────────────────┼──────────────────────┤
│ 欧氏距离      │ 绝对位置            │ ★ 较少用              │
│              │ 受模长和方向影响    │ 适合聚类/可视化        │
└──────────────┴──────────────────┴──────────────────────┘
```

## 六、为什么 RAG 通常用余弦相似度

```python
# RAG场景的特点：
# 1. 文档长度不一（一句话 vs 一段落）
#    → Embedding模长不同
#    → 点积会偏好长文档
#    → 余弦归一化后公平比较

# 2. 我们关心"语义相似"（方向）
#    而非"语义强度"（模长）
#    → 余弦最合适

# 实践：Embedding后先归一化，然后用点积（=余弦但更快）
def normalized_embedding(vec):
    """归一化：模长变为1"""
    return vec / np.linalg.norm(vec)

# 归一化后：dot(normalized_a, normalized_b) == cosine(a, b)
# 但点积计算比余弦少两次开方，更快
```

## 七、相似度阈值

```python
# 实践中的阈值参考（余弦相似度）
thresholds = {
    "> 0.95": "几乎相同（重复/近义）",
    "0.85-0.95": "非常相似（同主题/同义）",
    "0.7-0.85": "相关（同领域）",
    "0.5-0.7": "弱相关",
    "< 0.5": "基本不相关",
}

# RAG中：
# 检索top-k后，可设阈值过滤：
relevant = [doc for doc in results if doc.similarity > 0.7]
# 过滤掉弱相关的，减少噪音
```

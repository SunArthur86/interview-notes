# 【神州专车面经】模型离线 AUC 高但线上效果差，查哪些问题？

## 一、五类根因排查框架

```
[1] 特征不一致（最常见，占 60%）
    │  线上特征 ≠ 离线特征
    ▼
[2] 数据分布不一致
    │  训练历史数据 vs 线上当下数据，分布漂移
    ▼
[3] 样本不一致
    │  训练正负采样 vs 线上真实分布
    ▼
[4] 评估目标不一致
    │  AUC 是排序指标 vs 业务看转化/收入
    ▼
[5] 工程链路问题
       特征延迟/模型版本错/缓存脏
```

## 二、特征不一致（最常见，重点查）

### 离线特征 vs 线上特征的差异源

| 差异类型 | 例子 |
|---------|------|
| **线上缺特征** | 离线用了"用户最近30天订单数"，线上实时算不出来 |
| **计算逻辑不同** | 离线用 SQL 算"7日活跃"，线上用 Flink 算，口径不一致 |
| **实时性不同** | 离线是 T+1 全量，线上是实时增量，同一特征值不同 |
| **特征工程不同** | 离线做了归一化/分箱，线上忘了做（或反过来） |
| **默认值不同** | 离线空值填 0，线上填 -1 |

### 怎么排查
```python
# 1. 线上打日志：把请求时的特征向量落表
# 2. 离线复算：用同一批请求的原始数据重新算特征
# 3. 对比：线上特征 vs 离线复算特征，找差异大的字段
SELECT request_id, feature_name, 
       online_value, offline_value,
       ABS(online_value - offline_value) AS diff
FROM feature_consistency_log
WHERE ABS(online_value - offline_value) > 0.01
ORDER BY diff DESC;
```

## 三、数据分布不一致（分布漂移）

```
训练数据：2025年6月的订单（历史）
线上数据：2026年6月的订单（当下）

可能漂移：
  - 季节性：6月 vs 12月，出行模式不同
  - 业务变化：新开了城市/新上了车型/改了定价
  - 外部环境：竞品补贴/政策变化/疫情
```

### 怎么排查
```python
# PSI（Population Stability Index）衡量分布稳定性
# PSI < 0.1 稳定，0.1-0.25 轻微漂移，> 0.25 显著漂移
def psi(train_dist, online_dist):
    return sum((online - train) * ln(online / train))

# 对每个特征算 PSI，找漂移大的
```

### 解法
- 定期重训（周/月级）
- 用近期数据做 fine-tune
- 在线学习（FTRL）实时适应

## 四、样本不一致

```
训练样本：正负采样后 1:10（负样本下采样）
线上样本：真实分布 1:1000（负样本远多）

问题：
  - 模型学到的概率是采样后的，不是真实概率
  - 校准没做（Platt Scaling / Isotonic Regression）
```

### 解法
- 采样后做概率校准
- 或用负样本权重（`sample_weight`）而非下采样

## 五、评估目标不一致

```
AUC：排序指标（把正样本排在负样本前面的概率）
业务指标：转化率/GMV/点击率/留存

不一致场景：
  - AUC 高但排序头部不准（业务只看 Top-K，AUC 看全局）
  - AUC 高但校准差（概率值偏离真实点击率）
  - AUC 高但业务目标不同（AUC 优化排序，业务要 GMV）
```

### 解法
- 看业务相关指标（Top-K Precision / NDCG / GAUC）
- 看校准指标（LogLoss / Brier Score）
- 用业务目标做评估（线上 A/B）

## 六、工程链路问题

| 问题 | 现象 | 怎么查 |
|------|------|--------|
| **特征延迟** | 实时特征算慢了，用旧值或默认值 | 看特征计算延迟 P99 |
| **模型版本错** | 线上加载的是旧模型 | 看模型版本日志 |
| **缓存脏** | 特征缓存没更新，用了过期值 | 看缓存 TTL 和更新逻辑 |
| **请求路由错** | A/B 实验分桶错，流量没到新模型 | 看实验配置和分桶日志 |

## 七、排查流程（结构化）

```
[1] 先查工程链路（最快排除）
    │  模型版本对吗？特征延迟正常吗？A/B 分桶对吗？
    ▼
[2] 查特征一致性
    │  线上特征落表，离线复算对比，找差异
    ▼
[3] 查分布漂移
    │  算特征 PSI，看是否漂移
    ▼
[4] 查样本一致性
    │  采样比例对吗？概率校准了吗？
    ▼
[5] 查评估目标
    │  看业务相关指标（不只 AUC）
```

## 八、加分点

- 说出 **特征一致性是最常见原因**，占 60% 以上
- 说出 **GAUC（Group AUC）**：按用户分组算 AUC，比全局 AUC 更贴合推荐/排序场景
- 说出 **离线在线一致性管道**：每天跑一致性校验任务，特征 diff 超阈值告警

## 九、雷区

- ❌ "AUC 高就上线" → 必须配线上 A/B + 一致性监控
- ❌ 只查模型不查特征 → 80% 问题在特征
- ❌ 忽略工程链路 → 明明模型没问题，是链路 bug

## 十、扩展

- **特征监控平台**：监控每个特征的分布（均值/方差/缺失率）+ PSI，漂移告警
- **影子模式（Shadow Mode）**：新模型线上不真正服务，只打日志预测，对比老模型效果
- **Multi-task Learning**：同时优化 AUC 和业务目标，避免目标不一致

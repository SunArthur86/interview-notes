# Agent 简历项目怎么找到可以深挖的点？

## 一、什么是"可深挖的点"

```
不可深挖（CRUD，面试官不感兴趣）：
  ✗ "用LangChain搭了个Agent"
  ✗ "集成了OpenAI API"
  ✗ "实现了对话功能"
  ✗ "用了向量数据库"

可深挖（有难点，面试官想问）：
  ✓ "检索召回率从60%优化到85%"
  ✓ "工具调用准确率从70%提升到92%"
  ✓ "单任务Token成本降低70%"
  ✓ "P99延迟从5s优化到1s"
  ✓ "搭建了评估闭环，持续优化"
  ✓ "解决了多用户记忆隔离问题"
```

## 二、找深挖点的方法

```
┌──────────────────────────────────────────────────┐
│            五类可深挖的点                            │
├──────────────────────────────────────────────────┤
│                                                    │
│  1. 准确率类（效果优化）                            │
│     - 检索召回率/精度提升                          │
│     - 工具调用准确率                              │
│     - 幻觉率降低                                  │
│     - 任务完成率提升                              │
│                                                    │
│  2. 性能类（速度优化）                              │
│     - 延迟降低                                    │
│     - 吞吐量提升                                  │
│     - 并发能力增强                                │
│                                                    │
│  3. 成本类（省钱）                                  │
│     - Token消耗降低                              │
│     - API成本优化                                │
│     - 模型路由省钱                                │
│                                                    │
```

## 三、深挖点的表达结构（STAR + 数据）

```markdown
# 好的简历条目示例

## ❌ 不好（太笼统）
"开发智能客服Agent，支持自动回答用户问题"

## ✅ 好（深挖+量化）
"智能客服Agent（日活1000+）
- **问题**：FAQ匹配准确率仅60%，大量转人工
- **方案**：设计向量+BM25+Rerank三路混合检索，
  HyDE查询改写解决语义鸿沟
- **结果**：准确率提升至88%，转人工率从40%降至18%
- **技术**：LangChain + Milvus + bge-reranker"
```

## 四、找深挖点的提问法

```
问自己这些问题，答案就是深挖点：

1. "这个项目最难的部分是什么？"
   → 难点就是深挖点

2. "哪些地方花了最多时间调优？"
   → 调优的过程是深挖点

3. "上线后遇到什么问题？怎么解决的？"
   → 线上问题是好素材

4. "和同类产品比，你的优势在哪？"
   → 差异化优势

5. "如果重做，你会怎么改进？"
   → 反思体现深度

6. "有哪些trade-off决策？"
   → "为什么用A不用B"是面试官爱问的
```

## 五、常见 Agent 项目的深挖点

```
项目类型              可深挖点
─────────────────────────────────────────────
RAG知识库            召回率/精度优化、分块策略、混合检索
客服Agent            FAQ匹配、意图识别、转人工策略
编程Agent            代码生成准确率、测试验证、错误恢复
数据分析Agent        Text2SQL准确率、图表生成、多轮分析
多Agent系统          协作架构、冲突避免、成本控制
文档处理Agent        长文档处理、信息提取、格式转换
```

## 六、简历深挖的"避坑"

```
❌ 避免的写法：
  - "负责Agent开发"（太泛）
  - "使用LLM技术"（没说做了什么）
  - "提升了效率"（没数据）
  - 堆砌技术名词（没说怎么用的）

✅ 推荐的写法：
  - 动词开头（设计/优化/实现/解决）
  - 有具体问题（"召回率低""延迟高"）
  - 有量化结果（"从X到Y""提升Z%"）
  - 有技术细节（具体用什么方案）
  - 有反思（"如果重做会..."）
```

## 七、面试加分点

1. **找"非 CRUD"的点**：面试官不在乎你用了什么框架，在乎你解决了什么难问题
2. **量化一切**：有数字的简历比没数字的强 10 倍
3. **准备反思**：面试官爱问"如果重做你会怎么改"——有反思说明真理解了

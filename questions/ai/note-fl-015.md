# 【字节飞连面经】SFT vs RLHF vs DPO：做什么 / 数据形态 / 难点

## 一、三阶段对比

| 阶段 | 做什么 | 数据形态 | 难点 |
|------|--------|---------|------|
| **SFT** | 监督微调，让模型学会"指令→答案"的格式 | (prompt, response) | 答案质量上限决定模型上限 |
| **RLHF** | 用人类偏好训 Reward Model，再用 PPO 优化 | (prompt, response_A, response_B, 谁更好) | 训练不稳定、超参敏感、贵 |
| **DPO** | 直接用偏好数据优化策略，**跳过 RM** | 同 RLHF 偏好对 | 稳定、便宜、效果接近 PPO |

## 二、完整训练流程

```
[1] Pretrain（预训练）
    │  海量无标注文本，next token prediction
    │  学会语言能力（续写）
    ▼
[2] SFT（监督微调）
    │  人工标注 (指令, 高质量答案)
    │  学会"指令跟随"格式
    ▼
[3] 偏好优化（RLHF 或 DPO）
    │  人工标注 (指令, 好答案, 坏答案)
    │  学会"哪个回答更好"
    │  → helpful / harmless / honest
```

## 三、为什么 SFT 后还要偏好优化

**SFT 只会"模仿"，不会"排序"**：
- SFT 学的是"给定指令，输出类似标注答案的内容"
- 但**不会区分"哪个回答更好"**
- 比如两个回答都格式正确，但一个更 helpful、一个更啰嗦——SFT 分不出来

**偏好优化让模型学到 3H**：
- **Helpful**：真正解决问题
- **Harmless**：不输出有害内容
- **Honest**：不编造（减少幻觉）

这些是排序信号，SFT 给不了。

## 四、RLHF 怎么做（已被 DPO 替代但要知道）

```
[1] 训 Reward Model（RM）
    │  人工标注 (prompt, response_A, response_B, A比B好)
    │  训一个分类器：给 (prompt, response) 打分
    ▼
[2] PPO 优化策略
    │  让 LLM 生成回答 → RM 打分 → 用分数做强化学习
    │  最大化 RM 分数 + KL 惩罚（防止偏离 SFT 模型太远）
```

**RLHF 的痛点**：
- 训练不稳定（PPO 超参敏感）
- 需要 4 个模型同时跑（policy / ref / reward / value）→ 显存爆炸
- RM 容易被 reward hacking（模型钻 RM 漏洞刷分）

## 五、DPO 怎么做（工业主流）

```
直接用偏好数据优化策略，跳过 RM：

  L_DPO = -log σ(β·[log π(y_win|x)/π_ref(y_win|x) 
                     - log π(y_lose|x)/π_ref(y_lose|x)])
```

**核心思想**：数学上可以证明，最优策略可以用偏好数据直接推导，不需要显式训 RM。DPO 把 RLHF 的两步（训RM+PPO）合并成一步（直接用偏好对训策略）。

**DPO 优势**：
- 稳定（无 PPO 的方差问题）
- 便宜（只需 2 个模型：policy + ref）
- 效果接近 PPO

## 六、加分点

- 说出 **DPO 数学推导的关键**：通过 Bradley-Terry 模型，把"最优 RM"和"最优策略"建立解析关系，从而跳过显式 RM
- 说出 **变体**：
  - **ORPO**：把 SFT 和偏好优化合并成一步（不需要单独 SFT）
  - **KTO**：只需要（prompt, response, 好坏标签），不需要成对偏好数据
  - **SimPO**：去掉 ref model，用长度归一化，更省显存

## 七、扩展

- **RM 标注一致性**：多人标注偏好会有分歧，用 Cohen's Kappa 衡量一致性，低一致性的数据要剔除
- **Reward Hacking**：模型钻 RM 漏洞刷分（如输出 RM 偏好的特定词）→ 用 KL 惩罚 + 多 RM 集成缓解
- **RLAIF**（Constitutional AI）：用 AI 代替人工标注偏好（Anthropic 的方法），降低标注成本

## 记忆要点

- SFT 学格式模仿：数据是(指令, 答案)，难点在答案质量决定模型上限，但无法区分好坏。
- RLHF 学偏好排序：训 RM+PPO，能实现3H(有用无害诚实)，但4个模型同跑导致显存爆炸且不稳。
- DPO 主流替代：跳过显式训 RM，直接用偏好对数据一步优化策略，更稳定省钱。
- 加分项：说出 DPO 基于 Bradley-Terry 模型将最优 RM 与策略建立解析关系。

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：SFT 学"模仿"，偏好优化学"排序"。但 SFT 的标注答案本身就是"好答案"（人工选过的），模型模仿好答案不就等于学到"好"了吗？为什么还要单独学排序？**

因为 SFT 的 loss 只看"模型输出和标注答案的交叉熵"，它把标注答案当唯一正确目标，不会比较"好答案 vs 差答案"。举例：同一指令下，回答 A（精准简洁）和 B（啰嗦但格式对）都可能是 SFT 的标注对象，SFT 模仿其中任一时学不到"A 比 B 好"这个偏好信号。更关键的是 helpful/harmless 这类维度——有害回答可能格式正确、语言流畅，SFT 如果标了它会学；偏好优化通过"好回答 vs 有害回答"的对比，让模型显式学到"有害的更差"。排序信号的本质是"相对比较"，SFT 的"绝对模仿"给不了。这也是为什么只做 SFT 的模型容易输出有害或幻觉内容——它没学过"不输出比输出更好"。

### 第二层：证据与定位

**Q：你做了一轮 DPO 训练，评测集上"好回答胜率"从 60% 提到 75%。你怎么确认是 DPO 真的生效了，而不是评测集偏差（评测集和训练集重叠）？**

做严格的 train/test 隔离和交叉验证。一是确保 DPO 的训练偏好数据和评测集没有 prompt 重叠（按 prompt 去重检查），有重叠的部分从评测集剔除。二是跑"held-out"评测——保留一部分偏好数据不进训练，只在评测时用，看模型在这些"没见过"的 prompt 上的胜率是否也提升。如果 held-out 胜率也到 75%，证明学到了泛化的偏好而非记忆。三是对比 SFT-only baseline（不经过 DPO 的同尺寸模型）在同一评测集的胜率，如果 SFT baseline 只有 60% 而 DPO 到 75%，差距归因于 DPO。四是用多 judge（不同模型做裁判）交叉评分，避免单 judge 的偏好偏差导致虚高。

### 第三层：根因深挖

**Q：RLHF 你说"4 个模型同跑导致显存爆炸"。但 4 个模型里 reference 和 value 模型是可以冻结的，真正训练的只有 policy 和 reward。为什么显存还是爆？**

因为"冻结"只是不更新梯度，前向计算仍然要加载模型权重和激活值。4 个模型（policy 训练、ref 冻结但前向、reward 冻结但前向、value 训练）的权重都要在显存里，即使两个冻结的不要梯度，它们的激活值（用于算 loss）也要存。一个 7B 模型权重约 14GB（FP16），4 个就是 56GB，加上训练用的优化器状态（Adam 的 momentum + variance，2 倍参数量）、梯度、激活值，轻松超过单卡 80GB。DPO 把它降到 2 个模型（policy 训练 + ref 冻结前向），显存减半，且没有 PPO 的 value model 和 reward model。这是 DPO 在工程上取代 PPO 的核心原因——不是数学上更优，而是"显存和稳定性"的工程胜利。进一步优化如 SimPO 连 ref model 都去掉（用长度归一化替代），显存再降。

**Q：那如果 DPO 效果接近 PPO 又更稳定，为什么还要研究 PPO？为什么不彻底放弃 PPO？**

因为 DPO 有已知局限：一是它假设偏好是"成对可比较"的，对"两个回答都好但风格不同"或"取决于上下文"的复杂偏好建模弱；二是 DPO 容易过拟合偏好数据（它直接拟合偏好对，没有 PPO 的在线生成 + RM 评估的探索），训练太久模型会"只学好坏判断但生成质量下降"；三是 PPO 能做"DPO 做不到的在线学习"——policy 生成新回答、RM 实时打分、policy 据此调整，这个闭环让模型探索分布外的回答，DPO 只能学静态数据集。研究 PPO（及其变体如 GRPO）是为了处理 DPO 搞不定的场景（如数学推理的强化学习，需要过程奖励而非结果偏好）。工业界对话场景 DPO 够用，但前沿研究（如 o1 类推理模型）仍需 PPO 式的在线 RL。

### 第四层：方案权衡

**Q：DPO 需要"成对偏好数据"（prompt, 好回答, 坏回答）。但人工标注成对数据很贵（要为同一 prompt 生成两个回答再比较）。为什么不直接用"好/坏标签"的单回答数据（KTO 的方式），不是更省标注成本吗？**

KTO（只需要 prompt + response + 好/坏标签）确实更省标注成本——不需要为同一 prompt 找两个回答配对。但成对数据的信息量更大：成对比较控制了"prompt 的难度方差"（同一 prompt 下比较，消除了"难 prompt 的回答普遍差"的干扰），而 KTO 的单回答标签受 prompt 难度干扰（一个差回答可能是 prompt 难，不是回答本身差）。所以 DPO 用成对数据训练信号更纯净，KTO 虽省成本但信号噪声大。选型看数据条件：如果已有大量成对偏好数据（如用户反馈"A 比 B 好"），用 DPO；如果只有"这个回答好/差"的单标签（如点赞/点踩），用 KTO。两者不是谁替代谁，而是匹配不同数据形态。实践中可以混合：成对数据走 DPO 主训，单标签数据走 KTO 补充。

**Q：RLAIF（用 AI 代替人工标注偏好）能大幅降本。但 AI judge 自己有偏差（偏好长回答、偏好自己生成的内容），用它标注的偏好数据训出来的模型会不会学到这些偏差？**

会，这是 RLAIF 的核心风险。AI judge 的偏差会被"蒸馏"进训练数据，最终模型学到"AI judge 认为好的"而非"人类认为好的"。缓解方法：一是校准 judge——用 swap position（交换好/坏回答的位置，看 judge 是否改变判断）和 multi-judge（多个不同模型投票）减少单一偏差；二是混合人工标注——核心场景（如安全/有害判断）仍用人工标，AI 只标低风险场景（如哪个回答更详细）；三是 Constitutional AI 思路——给 AI judge 一套"宪法"（明确的评判标准如有害性、诚实性、helpfulness 的定义），让判断基于规则而非模型直觉。完全用 RLAIF 仍需验证：定期抽样 AI 标的偏好对让人工复核，一致率低于阈值（如 80%）说明 judge 偏差过大，要调 prompt 或换 judge 模型。RLAIF 是"成本和质量的权衡"，不是"免费午餐"。

### 第五层：验证与沉淀

**Q：你怎么证明 SFT→DPO 这套流程训出来的模型比"只 SFT"更好？用哪些指标？**

三层指标。一是自动评测：在公开 benchmark（如 AlpacaEval、MT-Bench、Arena-Hard）上跑 SFT-only vs SFT+DPO 的胜率/分数，DPO 版应显著高（如 AlpacaEval 胜率从 50% 到 70%）。二是对抗评测：专门测 3H——helpful（任务完成率）、harmless（有害内容拒绝率，用 AdvBench 等红队数据集）、honest（幻觉率，用 TruthfulQA），DPO 版应在 harmless 和 honest 上明显提升（这是偏好优化的核心价值）。三是人工评测：盲测中让人对比 SFT-only 和 SFT+DPO 的回答，DPO 版胜率应 > 60%。关键是要有 SFT-only 的 baseline 对比，不能只报 DPO 版的绝对分数——没有 baseline 无法归因"提升来自 DPO 还是 SFT 本身"。

**Q：怎么让团队在模型训练流程上（SFT→DPO 的数据质量、超参选择）保持一致，而不是每次训练结果不可复现？**

把训练流程和数据规范工程化。一是数据规范：SFT 答案的标注标准（格式、长度、风格）写成文档并配范例；偏好数据的标注要算 Cohen's Kappa（多人标注一致性），低于 0.6 的数据剔除。二是超参模板：DPO 的关键超参（β/KL 情罚系数、学习率、训练轮数）按模型尺寸固定一套基线配置，变更要记录理由和效果对比。三是训练日志：每次训练记录数据版本（hash）、超参配置、评测分数，可追溯可复现——出问题能定位是"数据变了"还是"超参变了"。四是回归评测：固定评测集（frozen），每次训练后跑同一套评测，对比历史分数，退化的实验不发布。让训练是"可控的实验科学"而非"炼丹"，靠的是数据/超参/评测的版本化和基线对比。

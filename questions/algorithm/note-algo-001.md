# 动态规划解题框架？怎么判断一道题该用DP？

## 一、什么情况下应该想到用 DP

动态规划（Dynamic Programming）本质上是一种**用空间换时间**的算法策略，适合解决具有以下三个特征的题目。在面试中，一旦题目出现了这三个信号，就应该优先考虑 DP。

### 1. 最优子结构

> 问题的最优解，可以由其子问题的最优解推导而来。

经典例子：LeetCode 53 最大子数组和。`dp[i]`（以 `nums[i]` 结尾的最大子数组和）的最优值，可以从 `dp[i-1]` 的最优值推导。

**反例**：如果子问题的最优解组合起来不等于全局最优（比如某些需要全局贪心的题），就不满足最优子结构。

### 2. 重叠子问题

> 同一个子问题会被反复计算。

这是 DP 与分治法的核心区别。归并排序的子问题不重叠（每次对半切，互不依赖），所以用分治；而斐波那契数列 `fib(5)` 会反复计算 `fib(3)`、`fib(2)`，子问题高度重叠，所以用 DP。

| 特征 | 分治法 | 动态规划 |
|------|--------|----------|
| 子问题重叠 | ❌ 不重叠 | ✅ 大量重叠 |
| 典型场景 | 归并/快排 | 斐波那契/背包 |
| 是否需要备忘录 | 不需要 | 需要（dp 数组） |

### 3. 无后效性

> 一旦当前状态确定，未来的决策只依赖当前状态值，不关心是怎么到达这个状态的。

如果题目有「未来的选择受历史路径影响」的需求（比如某些带路径约束的图问题），通常需要扩展状态维度来消除后效性，而不是直接不能用 DP。

**速判口诀**：求最值（最大/最小/最长/最短）+ 能拆分子问题 + 子问题会重复算 → 90% 是 DP。

## 二、DP 解题五步法

这是面试中最通用的 DP 解题框架，按顺序执行即可。

### Step 1：定义状态（最关键！）

状态定义直接决定后续转移方程的复杂度。面试中常见两种套路：

- **`dp[i]` = 考虑前 i 个元素时的最优值**（一维序列型：爬楼梯、打家劫舍）
- **`dp[i][j]` = 两个序列分别考虑前 i / 前 j 个时的最优值**（双序列型：LCS、编辑距离）

> 💡 状态定义不唯一！如果转移方程推不出来，换一个状态定义往往豁然开朗。

### Step 2：推导状态转移方程

找出 `dp[i]` 与之前状态的关系。常见模式：

```
线性递推:  dp[i] = dp[i-1] + dp[i-2]        （爬楼梯）
选择型:    dp[i] = max(dp[i-1], dp[i-2]+nums[i])  （打家劫舍）
区间型:    dp[i][j] = min(dp[i][k] + dp[k][j])    （戳气球）
```

### Step 3：确定初始值（Base Case）

最小子问题的答案，是递推的起点。必须让所有被引用的初始状态都有合理值。

### Step 4：确定遍历顺序

核心原则：**计算 `dp[i]` 时，它依赖的所有状态必须已经算好。**

- 一维 DP：通常从左到右
- 二维 DP：`dp[i][j]` 依赖 `dp[i-1][j]` 和 `dp[i][j-1]` → 从上到下、从左到右
- 区间 DP：先枚举区间长度（从小到大），再枚举起点

### Step 5：空间优化

如果 `dp[i]` 只依赖前面有限个状态（如 `dp[i-1]`、`dp[i-2]`），可以用**滚动变量**或**滚动数组**把 O(n) 空间降到 O(1)。

## 三、实战示例：打家劫舍（LeetCode 198）

### 题目

给定一个非负整数数组 `nums`，不能选相邻元素，求能抢到的最大金额。

### 五步法落地

| 步骤 | 内容 |
|------|------|
| 状态定义 | `dp[i]` = 前 i 个房子能抢到的最大金额 |
| 转移方程 | `dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])` |
| 初始值 | `dp[0] = 0; dp[1] = nums[0]` |
| 遍历顺序 | i 从 2 到 n |
| 空间优化 | 滚动两个变量 |

### Python 代码

```python
def rob(nums: list[int]) -> int:
    if not nums:
        return 0
    prev2, prev1 = 0, nums[0]  # dp[i-2], dp[i-1]
    for i in range(1, len(nums)):
        curr = max(prev1, prev2 + nums[i])
        prev2, prev1 = prev1, curr
    return prev1
```

### Java 代码

```java
public class HouseRobber {
    public int rob(int[] nums) {
        if (nums == null || nums.length == 0) return 0;
        int prev2 = 0, prev1 = nums[0]; // dp[i-2], dp[i-1]
        for (int i = 1; i < nums.length; i++) {
            int curr = Math.max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}
```

## 四、经典 DP 模型速查表

| 模型 | 状态含义 | 典型题目 | 难度 |
|------|----------|----------|------|
| 一维线性 DP | `dp[i]` = 前 i 项最优 | 爬楼梯、打家劫舍、最长递增子序列 | Easy~Medium |
| 二维 DP | `dp[i][j]` = 双序列匹配 | LCS、编辑距离、最长回文子序列 | Medium |
| 背包 DP | `dp[i][j]` = 前 i 物品容量 j | 0-1 背包、完全背包、零钱兑换 | Medium |
| 区间 DP | `dp[i][j]` = 区间 [i,j] | 戳气球、矩阵链乘、回文分割 | Hard |
| 树形 DP | `dp[node]` = 以 node 为根 | 打家劫舍 III、二叉树最大路径和 | Medium~Hard |
| 状态压缩 DP | `dp[mask]` = 选取集合 | 旅行商问题、任务分配 | Hard |

## 五、从暴力递归到动态规划的演进路径

面试中推荐按以下顺序推导，思路最清晰：

```
暴力递归（指数级）
    │  发现重叠子问题
    ▼
递归 + 备忘录（自顶向下，记忆化搜索）
    │  改为自底向上
    ▼
递推 DP（自底向上，dp 数组）
    │  状态依赖有限
    ▼
滚动数组 / 滚动变量（空间优化）
```

### 示例：斐波那契数列的三种写法

```python
# ① 暴力递归 O(2^n)
def fib(n):
    if n <= 1: return n
    return fib(n - 1) + fib(n - 2)

# ② 记忆化搜索 O(n) 自顶向下
from functools import lru_cache
@lru_cache(maxsize=None)
def fib(n):
    if n <= 1: return n
    return fib(n - 1) + fib(n - 2)

# ③ 递推 DP O(n) 自底向上 + 空间优化
def fib(n):
    if n <= 1: return n
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b
```

记忆化搜索 vs 递推对比：

| 维度 | 记忆化搜索（自顶向下） | 递推 DP（自底向上） |
|------|----------------------|-------------------|
| 思维方式 | 从大问题拆小，自然 | 从小问题推大 |
| 空间 | 递归栈开销 | 可优化到 O(1) |
| 适用场景 | 状态稀疏时更优 | 状态密集时更优 |
| 面试推荐 | 推导阶段使用 | 最终代码使用 |

## 六、面试高频追问

### Q1：DP 和贪心的区别？

- **贪心**：每一步选当前最优，不回退。要求局部最优能推导全局最优（贪心选择性）。
- **DP**：枚举所有子问题的最优，再组合。更通用但复杂度更高。
- **判断**：如果贪心策略能严格证明正确，用贪心（更快）；否则用 DP（更保险）。

### Q2：记忆化搜索和递推哪个好？

- 记忆化搜索**思路自然**，适合状态定义复杂或状态空间稀疏的场景（如某些博弈 DP）。
- 递推 DP**效率更高**，无递归开销，空间可优化，是面试最终代码的首选。
- 实战中常先用记忆化搜索验证正确性，再转为递推优化性能。

## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：DP 三大特征里，"无后效性"你怎么向一个没学过 DP 的人解释？为什么它是一票否决项？**

无后效性指"当前状态一旦确定，未来怎么走只看这个状态值，不在乎你是怎么来的"。给新人举斐波那契：fib(5) 只取决于 fib(4)+fib(3)，它不关心你是先算 fib(3) 还是先算 fib(2)。之所以是一票否决项，是因为 DP 的状态转移方程 `dp[i] = f(dp[j])` 隐含假设：`dp[i]` 是子问题的完整描述。如果未来还依赖"到达 dp[i] 的具体路径"，那 `dp[i]` 这一个数就不足以支撑后续决策，方程根本写不出来——必须把路径信息并入状态（如把 `dp[i]` 升级成 `dp[i][j]`），否则只能放弃 DP 换搜索。

### 第二层：证据与定位

**Q：面试中拿到一道求最值的题，你怎么在 30 秒内判断"该用 DP 还是贪心"？**

两条证据交叉判断：一看子问题是否独立——如果每一步的局部最优能滚出全局最优（如找零的某些面值、区间调度按结束时间排序），用贪心；二看子问题是否重叠——如果同一个子状态会被多条路径反复访问（如背包、最长公共子序列），用 DP。具体到代码：贪心每步 O(1) 决策不回头，DP 是把所有子状态填满一张表再回推。30 秒判定法：试着写暴力递归树，如果树里出现大量重复节点（重叠子问题）且不能贪心（局部最优≠全局最优），就是 DP。

### 第三层：根因深挖

**Q：你写了一个二维 DP，结果 OJ 上 5/10 用例超时。你怎么定位是状态定义错了，还是转移方程写错了？**

先打印小规模 case 的 dp 表肉眼检查。如果表里某些值明显不对（比如 dp[i][j] 应该单调却出现回退），是转移方程 bug——重点查 `for` 循环的遍历顺序（比如背包问题先遍历容量还是先物品、容量是正序还是逆序决定了是 0-1 还是完全背包）。如果表值都合理但整体偏大，是状态定义太粗——典型例子：LeetCode 123 买卖股票最多 2 次交易，如果只定义 `dp[i]` 表示第 i 天最大利润，无法表达"当前是否持有 + 已交易几次"，必须扩成 `dp[i][k][0/1]`。状态维度不够会导致子问题信息丢失，方程再对也错。

**Q：那为什么不直接把状态维度堆到很高，反正维度越高信息越全？**

维度越高状态空间越大，复杂度爆炸。三维 DP `O(n^3)` 在 n=500 时就是 1.25 亿次运算，会超时；四维以上几乎只适用于 n≤50 的小数据。堆维度的本质是把"未来需要的所有历史信息"压进状态，正确做法是只加"消除后效性所必需的那一维"——比如买卖股票的交易次数 k 是必须的（因为题目约束了最多 K 次），但"昨天是不是周一"这种无关信息绝不加。维度取舍是用题目约束反推的，不是越多越好。

### 第四层：方案权衡

**Q：状态转移方程推出来后，空间优化你怎么决定要不要做？**

看空间复杂度是否是瓶颈，不是无脑优化。规则：一维 dp 且依赖 `dp[i-1]` → 滚动数组降到 O(1)；二维背包如果只依赖"上一层" → 用一维数组逆序遍历降空间但代码更易错（逆序写错就变完全背包）。权衡点：空间优化常牺牲可读性，且调试时看不到完整 dp 表很难排查。我的做法是先写不优化的版本过 OJ，再单独提一个 commit 做滚动数组优化，并用同一组测试用例对拍验证答案一致。

**Q：为什么不直接用记忆化搜索（递归+备忘录），代码不是更直观吗？**

记忆化搜索确实更接近自然思维，但它有两个硬伤：一是递归栈深度受限于语言（Python 默认 1000，Java 约 1 万），状态空间大时直接 StackOverflow；二是常数因子大（函数调用开销 + 哈希表查表），同样的算法递推能比记忆化快 3-5 倍。所以面试标准流程是：先用记忆化搜索验证思路正确性（写起来快、不易写错遍历顺序），再改写成递推 DP 提交（性能可控、空间可优化）。两者是"验证→优化"的关系，不是二选一。

### 第五层：验证与沉淀

**Q：你怎么验证自己写的 DP 一定是对的，而不只是碰巧过了样例？**

三步验证：一是边界 case——dp[0]、dp[1]、空数组、单元素、全负数、全相同这些极端输入单独构造测试；二是对拍——写一个暴力 DFS（哪怕 O(2^n)）在小数据（n≤15）上跑，跟 DP 结果逐个对比，跑几千组随机数据全对才算稳；三是断言不变式——比如最长递增子序列的 dp[i] 应满足"dp 数组的前若干项是单调不降的（如果用 patience sorting 优化版）"，在代码里加 assert 检验。样例只覆盖 happy path，对拍才能抓到转移方程的隐蔽 bug。

**Q：这道题做完，你沉淀出了什么可复用的解题模板？**

DP 解题五步法固化成检查清单：① 状态定义（dp[i] / dp[i][j] 代表什么具体含义，必须一句话写清楚）；② 转移方程（从哪些子状态来、取 max/min/sum）；③ 初始值（dp[0]=？，避免用 dp[-1] 这种越界）；④ 遍历顺序（依赖谁就先遍历谁，背包要分清先容量还是先物品）；⑤ 空间优化（看依赖关系决定能否滚动）。这套模板我已经用于 LeetCode 50+ 道 DP 题的复习，每次卡壳回头看模板，80% 的情况是卡在第 1 步（状态定义不够细）。

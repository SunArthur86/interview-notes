# 冒泡排序的时间复杂度？

## 算法核心

```
原始数组: [5, 3, 8, 1, 9, 2]

第1轮：相邻比较交换，最大的冒到最后
[5,3,8,1,9,2] → [3,5,8,1,9,2] → [3,5,1,8,9,2] → [3,5,1,8,9,2] → [3,5,1,8,2,9]
                                                                              ↑ 9到位

第2轮：
[3,5,1,8,2,9] → [3,1,5,8,2,9] → [3,1,5,2,8,9]
                                              ↑ 8到位

第3轮：
[3,1,5,2,8,9] → [1,3,5,2,8,9] → [1,3,2,5,8,9]
                                    ↑ 5到位

第4轮：
[1,3,2,5,8,9] → [1,2,3,5,8,9]
                ↑ 3到位

第5轮：
[1,2,3,5,8,9] → 无交换 → 已排序 ✅
```

## 代码实现

### 基础版本

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1):           # n-1轮
        for j in range(n - 1 - i):   # 每轮比较n-1-i次
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr
```

### 优化版本（提前终止）

```python
def bubble_sort_optimized(arr):
    """加入flag，已排序时提前终止 → 最好O(n)"""
    n = len(arr)
    for i in range(n - 1):
        swapped = False  # 本轮是否发生交换
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        
        if not swapped:  # 本轮无交换 → 已排序
            break
    return arr
```

## 时间复杂度分析

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| **最好** | O(n) | 数组已排序，优化版第一轮就终止 |
| **平均** | O(n²) | 比较次数 = n(n-1)/2 |
| **最坏** | O(n²) | 逆序数组，每次都要交换 |

### 推导过程

```
比较次数 = (n-1) + (n-2) + ... + 1 = n(n-1)/2

n=10:  45次比较
n=100: 4950次比较  
n=1000: 499500次比较

O(n²)
```

## 空间复杂度

```
O(1) — 原地排序，只用常数级额外空间
```

## 稳定性

```
稳定排序 ✅

相等元素不交换：
[3a, 3b, 1] → [3a, 1, 3b] → [1, 3a, 3b]
               ↑              ↑
          3a和3b不交换    3a仍在3b前面
```

## 冒泡排序 vs 其他排序

| 排序 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| **冒泡** | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 选择 | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| 插入 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 快排 | O(nlogn) | O(nlogn) | O(n²) | O(logn) | ❌ |
| 归并 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | ✅ |

## 面试加分点

1. **最好O(n)的条件**：加了swapped flag优化后，已排序数组只需一轮
2. **稳定排序**：相等元素相对位置不变，这在多关键字排序时很重要
3. **与选择排序区别**：冒泡每轮可能多次交换，选择每轮只交换一次
4. **实用价值低**：O(n²)排序中插入排序实际表现通常优于冒泡

- 空间复杂度O(1)，因为是原地交换只用常数级额外空间。

- 对比选择排序：冒泡每轮可多次交换且稳定，而选择每轮只交换一次且不稳定。

# 手撕：力扣56.合并区间

## 题目

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]`。合并所有重叠的区间，返回一个不重叠的区间数组。

## 图解

```
输入: [[1,3],[2,6],[8,10],[15,18]]

Step 1: 按起点排序 (已排序)
  [1,3] [2,6] [8,10] [15,18]

Step 2: 遍历合并
  ┌─────┐
  │[1,3]│  ← 初始化result=[[1,3]]
  └──┬──┘
     │ [2,6].start=2 <= 3=current_end → 重叠!
     ▼
  ┌──────┐
  │[1,6] │  ← end = max(3, 6) = 6
  └──┬───┘
     │ [8,10].start=8 > 6=current_end → 不重叠
     ▼
  ┌──────┐ ┌──────┐
  │[1,6] │ │[8,10]│  ← 直接入结果
  └──────┘ └──┬───┘
              │ [15,18].start=15 > 10 → 不重叠
              ▼
  ┌──────┐ ┌──────┐ ┌───────┐
  │[1,6] │ │[8,10]│ │[15,18]│
  └──────┘ └──────┘ └───────┘

输出: [[1,6],[8,10],[15,18]]
```

## 代码

```python
def merge(intervals: list[list[int]]) -> list[list[int]]:
    """
    合并重叠区间
    时间: O(n log n) - 排序
    空间: O(n) - 结果数组
    """
    if not intervals:
        return []
    
    # Step 1: 按起点排序
    intervals.sort(key=lambda x: x[0])
    
    # Step 2: 遍历合并
    result = [intervals[0]]
    
    for i in range(1, len(intervals)):
        curr = intervals[i]
        prev = result[-1]
        
        if curr[0] <= prev[1]:
            # 重叠 → 合并，取较大的end
            prev[1] = max(prev[1], curr[1])
        else:
            # 不重叠 → 直接入结果
            result.append(curr)
    
    return result

# 测试
print(merge([[1,3],[2,6],[8,10],[15,18]]))  # [[1,6],[8,10],[15,18]]
print(merge([[1,4],[4,5]]))                   # [[1,5]] (端点相等也算重叠)
print(merge([[1,4],[0,4]]))                   # [[0,4]] (排序后[0,4]在前)
```

## 边界情况

```python
# 1. 空数组
merge([])                              # []

# 2. 单个区间
merge([[1,1]])                         # [[1,1]]

# 3. 全部重叠
merge([[1,4],[2,3]])                   # [[1,4]] (包含关系)

# 4. 完全不重叠
merge([[1,2],[3,4],[5,6]])            # [[1,2],[3,4],[5,6]]

# 5. 未排序输入
merge([[2,6],[1,3],[8,10],[15,18]])   # [[1,6],[8,10],[15,18]]
```

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n log n) | 排序O(n log n) + 遍历O(n) |
| 空间 | O(n) | 结果数组(不算排序的O(log n)栈空间) |

## 变体题

```python
# 变体: 插入区间 (力扣57)
# 在已排序不重叠区间中插入一个新区间，返回合并后的结果

def insert(intervals, newInterval):
    """O(n)解法，无需排序(输入已排序)"""
    result = []
    i = 0
    n = len(intervals)
    
    # 1. 添加所有在newInterval之前的区间
    while i < n and intervals[i][1] < newInterval[0]:
        result.append(intervals[i])
        i += 1
    
    # 2. 合并所有与newInterval重叠的区间
    while i < n and intervals[i][0] <= newInterval[1]:
        newInterval[0] = min(newInterval[0], intervals[i][0])
        newInterval[1] = max(newInterval[1], intervals[i][1])
        i += 1
    result.append(newInterval)
    
    # 3. 添加剩余区间
    while i < n:
        result.append(intervals[i])
        i += 1
    
    return result
```

## 记忆要点

- 第一步必须排序：因为区间乱序无法直接比较，所以先按起点进行排序。
- 遍历条件判断：因为当前区间start <= 上一区间end意味着重叠，所以合并并取较大的end。
- 不重叠直接追加：若当前start > 上一区间end，则直接加入结果集。
- 复杂度记忆：排序耗时O(n log n)，单次遍历O(n)，总时间复杂度为O(n log n)。


## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：合并区间你为什么先排序再遍历，而不是两两比较所有区间对？**

两两比较是 O(n²)——n 个区间要比 n(n-1)/2 次，且合并后产生新区间又要重新比较，复杂度更高。排序的本质是"建立全局有序性"：按起点排序后，能合并的区间一定相邻，扫一遍就能合并完，O(n log n)+O(n)=O(n log n)。排序的价值不是"让数据好看"，而是把"任意两个区间可能重叠"的二维问题降维成"相邻区间才可能重叠"的一维问题。这是处理区间类问题（合并、插入、会议室）的通用第一步。

### 第二层：证据与定位

**Q：你说排序后可合并的区间一定相邻，怎么证明？有没有反例？**

证明：设排序后区间序列 I₁, I₂, ..., Iₙ（按 start 升序）。假设 Iᵢ 和 Iⱼ 重叠（i<j），即 Iᵢ.end ≥ Iⱼ.start。由于 start 单调，Iᵢ.start ≤ Iᵢ₊₁.start ≤ ... ≤ Iⱼ.start ≤ Iᵢ.end，所以 Iᵢ₊₁.start ≤ Iᵢ.end，即 Iᵢ 和 Iᵢ₊₁ 也重叠。同理 Iᵢ₊₁ 和 Iᵢ₊₂、... 直到 Iⱼ 都连成一片。所以"跳跃式重叠"不存在，重叠的区间必然连续。反例：如果排序键不是 start 而是 end，这个性质不成立——比如 [1,5] 和 [2,3]，按 end 排序是 [2,3],[1,5] 看起来不连续但实际重叠。所以必须按 start 排序。

### 第三层：根因深挖

**Q：面试官给你一个变体——插入新区间到已排序的区间列表里（LeetCode 57），你怎么处理？根因上和合并区间有什么区别？**

输入已经是排序好的，省了排序步骤，但新区间可能与多个已存在区间重叠。分三段处理：① 把所有 end < newInterval.start 的区间直接加入结果（在新区间左边，不重叠）；② 把所有 start ≤ newInterval.end 的区间与 newInterval 合并（重叠区间，反复取 min start / max end）；③ 剩余区间直接加入结果（在右边，不重叠）。根因区别：合并区间（56）是"全局无序需排序"，插入区间（57）是"已知有序利用有序"。两者本质都是"重叠判断 + 区间合并"，但 57 利用了输入的有序性把复杂度从 O(n log n) 降到 O(n)。

**Q：那为什么不直接用 56 题的解法——把新区间 append 进去再排序合并？**

能 AC 但浪费了"已有序"这个已知信息。57 题的输入 intervals 已经按 start 排好，直接 append + 重新排序是 O(n log n)，而三段扫描是 O(n)。面试场景下，面试官出 57 而不是 56 就是在考察"你能不能识别并利用输入的额外性质"。如果照搬 56 解法，相当于告诉面试官"我没注意到输入已经有序"，会扣分。工程上也是同理——已有索引的数据查询就别全表扫描，已有的不变式要充分利用，不要做无谓的重复劳动。

### 第四层：方案权衡

**Q：如果区间数量很大（百万级），你的 O(n log n) 排序可能是瓶颈，怎么优化？**

看区间值的分布。如果区间端点范围有限（比如时间戳在一天内、值是 int32 且密集），可以用桶排序或计数排序降到 O(n+max-min)。如果区间来自多个有序数据源（如多个机器的分片结果），可以用 k 路归并（堆维护 k 个指针），复杂度 O(n log k)。如果只是合并而不关心顺序，甚至可以并行化——分片各自合并，再归并结果。权衡点：通用排序（快排/归并）O(n log n) 是稳妥默认值，只有当 n 真的大到成瓶颈且数据有特殊结构时才上特殊排序，否则工程复杂度不划算。

**Q：为什么不直接用线段树或区间树，它们不是专门处理区间的吗？**

线段树/区间树解决的是"动态查询"——区间频繁增删 + 区间查询（如"查询与某点相交的所有区间"）。合并区间是"一次性离线处理"，不需要动态增删，建树的开销（O(n log n)）和空间（O(n)）反而比直接排序大。工具匹配原则：线段树适合在线查询场景（如会议室 II 求 K 个重叠区间），合并区间这种一次性 batch 处理用排序+扫描更直接。错用线段树会让代码从 20 行膨胀到 100 行，且性能没提升。

### 第五层：验证与沉淀

**Q：你怎么验证合并区间实现的正确性，特别是边界 case？**

四类必测：① 空列表 []→[]；② 单区间 [[1,1]]→[[1,1]]（退化区间）；③ 全部不相交 [[1,2],[3,4],[5,6]]→原样；④ 全部嵌套/相交 [[1,10],[2,5],[3,8],[6,9]]→[[1,10]]。重点测边界：start 相等但 end 不同（如 [1,3] 和 [1,5] 应合并成 [1,5]）、一个区间完全包含另一个、零长度区间 [a,a]。对拍验证：随机生成 1000 组区间，跟朴素 O(n²) 的合并算法（每次找任一可合并对合并，直到稳定）对比，全对才算稳。

**Q：这道题沉淀出了什么可复用的区间类问题解法？**

区间类问题三步法：① 按 start（或 end，看问题）排序；② 扫描时维护"当前合并结果"或"活动区间集合"；③ 重叠判断条件 `curr.end >= next.start` 触发合并。具体变体：合并区间（56）维护当前合并区间、插入区间（57）分三段处理、会议室（252）检查相邻是否冲突、会议室 II（253）用最小堆维护结束时间、用最少箭引爆气球（452）按 end 排序贪心。这一类题在 LeetCode 上有 10+ 道，套路高度一致，掌握排序+扫描的模板能批量秒杀。


## 结构化回答

**30 秒电梯演讲：** 先按起点排序，再遍历一次合并有重叠的区间。打个比方，就像整理时间表——先把所有会议按开始时间排序，然后从头到尾看，有重叠的会议合并成一个大的时间段。

**展开框架：**
1. **第一步必须排序** — 因为区间乱序无法直接比较，所以先按起点进行排序。
2. **遍历条件判断** — 因为当前区间start <= 上一区间end意味着重叠，所以合并并取较大的end。
3. **不重叠直接追加** — 若当前start > 上一区间end，则直接加入结果集。

**收尾：** 这块我踩过坑——要不要深入聊：如果要返回哪些区间有重叠(而不是合并)，怎么做？

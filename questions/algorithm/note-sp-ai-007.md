# 怎么用两个栈实现队列？

## 核心思路

```
队列操作: FIFO（先进先出）
栈操作: LIFO（后进先出）

两个栈串联 = 两次反转 = FIFO

入队 1, 2, 3:
┌─────────────┐    ┌─────────────┐
│  入队栈      │    │  出队栈      │
│  (push栈)   │    │  (pop栈)    │
│             │    │             │
│  [3] ← 顶   │    │  [ ] 空     │
│  [2]        │    │             │
│  [1] ← 底   │    │             │
└─────────────┘    └─────────────┘

出队时，出队栈空→倒灌：
┌─────────────┐    ┌─────────────┐
│  入队栈      │    │  出队栈      │
│  [ ] 空     │──→ │  [1] ← 顶   │
│             │ 倒灌│  [2]        │
│             │    │  [3] ← 底   │
└─────────────┘    └─────────────┘

出队栈弹出 → 得到1（FIFO顺序 ✅）
```

## 代码实现

```python
class MyQueue:
    def __init__(self):
        self.in_stack = []   # 入队栈
        self.out_stack = []  # 出队栈
    
    def push(self, x: int) -> None:
        """入队：直接压入入队栈"""
        self.in_stack.append(x)
    
    def pop(self) -> int:
        """出队：从出队栈弹出"""
        self._transfer()  # 确保出队栈有元素
        return self.out_stack.pop()
    
    def peek(self) -> int:
        """查队首：不弹出"""
        self._transfer()
        return self.out_stack[-1]
    
    def empty(self) -> bool:
        """是否为空"""
        return not self.in_stack and not self.out_stack
    
    def _transfer(self):
        """如果出队栈为空，把入队栈全部倒入"""
        if not self.out_stack:
            while self.in_stack:
                self.out_stack.append(self.in_stack.pop())
```

## 执行过程演示

```
操作序列: push(1), push(2), push(3), pop(), push(4), pop(), pop()

push(1): in=[1], out=[]
push(2): in=[1,2], out=[]
push(3): in=[1,2,3], out=[]

pop(): out空→倒灌: in=[], out=[3,2,1]
       弹出1 → 返回1
       in=[], out=[3,2]

push(4): in=[4], out=[3,2]

pop(): out不空→直接弹出2 → 返回2
       in=[4], out=[3]

pop(): out不空→直接弹出3 → 返回3
       in=[4], out=[]

最终: in=[4], out=[]
```

## 均摊O(1)分析

```
关键洞察：每个元素最多被搬运一次（从in_stack到out_stack）

push(1) → 1进入in_stack     搬运次数=0
倒灌     → 1从in到out        搬运次数=1  ← 只搬这一次
pop()    → 1从out弹出        搬运次数=0

总搬运次数 = N个元素各搬1次 = N次
总操作次数 = N次push + N次pop + N次搬运 = 3N

均摊到每次操作 = 3N / 2N = 1.5 → O(1)

虽然单次pop可能触发倒灌O(n)，但均摊到所有操作是O(1)
```

## 复杂度

| 操作 | 最好 | 最坏 | 均摊 |
|------|------|------|------|
| push | O(1) | O(1) | O(1) |
| pop | O(1) | O(n)* | O(1) |
| peek | O(1) | O(n)* | O(1) |
| empty | O(1) | O(1) | O(1) |

> *最坏情况：出队栈空时需要倒灌整个入队栈

## 面试加分点

1. **均摊分析**：每个元素只搬运一次，所以均摊O(1)
2. **倒灌时机**：只在out_stack为空时倒灌，避免反复搬运
3. **不变式**：out_stack中的元素已经是FIFO顺序，不需要每次倒灌
4. **负负得正**：LIFO + LIFO = FIFO，两次反转恢复原始顺序

## 记忆要点

- 负负得正：因为栈是LIFO，所以两个栈串联倒灌两次即可实现队列的FIFO。
- 关键时机：只有当出队栈为空时，才将入队栈全部倒入出队栈。
- 均摊O(1)：因为每个元素最多被搬运一次，所以平摊到每次操作复杂度为O(1)。


## 苏格拉底式面试追问

> 这组追问模拟面试官层层逼问，每一问先回答"为什么"，再回答"怎么做"，最后回答"如何证明"。

### 第一层：目标与动机

**Q：用两个栈实现队列，你说"负负得正——LIFO + LIFO = FIFO"，这个直觉背后的形式化证明是什么？**

栈是 LIFO（后进先出）。元素依次 push 1,2,3 到栈 A（入队栈），A 里从顶到底依次是 3,2,1。把 A 全部 pop 到栈 B（出队栈），B 从顶到底依次是 1,2,3——顺序被反转了两次（A 的 LIFO 反转一次 + pop 到 B 再反转一次），两次反转恢复原序，所以 B 的 pop 顺序是 1,2,3，即 FIFO。形式化：设入队序列是 e₁,e₂,...,eₙ，push 到 A 后 A 顶是 eₙ；pop 到 B 后 B 顶是 e₁；B.pop() 返回 e₁，与入队顺序一致。这就是队列语义。关键不变式：B 栈里的元素始终按 FIFO 顺序排列，只要 B 不空就直接 pop B。

### 第二层：证据与定位

**Q：你说"只有 B 为空时才把 A 倒入 B"，如果不满足这个条件（B 非空时也倒），会出什么问题？**

会破坏 FIFO 顺序。举例：push 1,2,3 到 A，倒到 B（B 顶是 1）；pop 返回 1（B 里剩 2,3）；这时再 push 4,5 到 A；如果立即把 A 倒入 B，B 里变成（从顶到底）5,4,2,3——下次 pop 返回 5 而不是 2，违反 FIFO。正确做法：B 里的元素是"已经排好 FIFO 序的待出队元素"，必须等 B 完全空了，才能把 A 的新元素倒入（重新反转），否则新旧元素顺序混乱。这个"B 空才倒"的条件是正确性的核心不变式，破坏它就破坏了队列语义。

### 第三层：根因深挖

**Q：你说每次操作均摊 O(1)，怎么证明？不是有 pop 触发 O(n) 的倒灌吗？**

用势能分析法（均摊分析）。每个元素的生命周期里只被搬运两次：push 时进 A（1 次）、倒灌时从 A 到 B（1 次），之后再不被搬运。N 次 push + N 次 pop 总搬运次数 ≤ 2N（每个元素最多搬 2 次），所以总工作量 O(N)，均摊到每次操作 O(2N/2N)=O(1)。虽然单次 pop 可能触发整个 A 倒入 B 的 O(n)（如 push n 个后第一次 pop），但这 O(n) 是"预付"了后续 n 次 pop 的搬运，后续 pop 都只需 O(1) 直接 pop B。关键：均摊不是平均，是"用大操作分摊到小操作"的严格分析。LeetCode 232 用这个分析能严格证明复杂度。

**Q：那为什么不能用一个栈实现队列，非得两个？**

一个栈不可能实现队列。栈的本质是 LIFO，单栈的所有操作都是 LIFO 语义，无法产生 FIFO 行为。直觉证明：队列要求"先入先出"，但栈顶永远是"最后入"的元素，单栈的 pop 永远返回最后入的，跟 FIFO 矛盾。两个栈之所以能实现，是因为"两次 LIFO 反转 = FIFO"，这是结构性的——必须有一个栈做"正序转反序"、另一个做"反序转正序"，少一个都无法完成两次反转。这是算法上的下界：实现队列至少需要两个栈（或一个能两头操作的 deque，但那就不是栈了）。

### 第四层：方案权衡

**Q：如果反过来——用两个队列实现栈，怎么做？跟两个栈实现队列哪个更优？**

两个队列实现栈：push 时把元素入空队列 Q1，再把另一队列 Q2 的元素全部出队到 Q1（保证新元素在队首），pop 直接出 Q1 队首。或者更简单的方案：push 直接入 Q1，pop 时把 Q1 前 n-1 个元素搬到 Q2，Q1 剩的最后一个就是栈顶，pop 后交换 Q1/Q2 引用。复杂度：一种 push O(n)、pop O(1)，另一种 push O(1)、pop O(n)。两个栈实现队列是均摊 O(1)，两个队列实现栈不能均摊 O(1)（每次 pop 都要搬运 n-1 个元素，没有"搬运一次后续免费"的性质）。所以"两个栈实现队列"比"两个队列实现栈"性能更好，这是由栈和队列的内在结构决定的。

**Q：为什么不直接用 ArrayDeque（双端队列），push/pop 两端操作，一个数据结构搞定？**

能，而且性能更好（ArrayDeque 两端操作都是 O(1)）。但题目要求"用栈实现队列"，限制就是只能用栈的 push/pop（单端操作），ArrayDeque 的双端操作超出了栈的语义，等于作弊。面试题的约束（"用两个栈"）是在考察"受限环境下的等价变换能力"，不是"找最快实现"。工程上当然直接用 ArrayDeque 或 LinkedList 当队列，面试题里要遵守约束。这个区别要分清——回答时可以说"生产用 ArrayDeque，但题目约束下我用两个栈这样实现"，体现工程+理论双重视角。

### 第五层：验证与沉淀

**Q：你怎么验证两个栈实现队列在各种操作序列下都正确？**

核心测两类序列：① 交叉操作 push 1,2,3 → pop → push 4 → pop → pop → pop，期望返回 1,2,3,4（测倒灌时机正确）；② 极端序列——连续 push n 个再连续 pop n 个（测批量倒灌）、push/pop 交替（测频繁倒灌）。重点验证"B 非空时 A 不能倒"这个不变式——构造"push 1,2 → pop（返回1）→ push 3 → pop（应返回2 不是3）"，错误实现会返回 3。对拍：用标准库 LinkedList 作为 oracle，随机生成 1000 次 push/pop 序列，两实现结果应完全一致。

**Q：这道题沉淀出了什么可复用的数据结构设计经验？**

两条可复用：一、"两次反转恢复原序"——这是用受限原语组合出新语义的通用技巧，比如"两个栈实现队列"、"用栈实现队列的反向迭代器"、"双端队列的中缀表达式求值"；二、"均摊分析"思维——不要被"单次最坏 O(n)"吓到，要分析每个元素的总搬运次数，判断均摊复杂度。这俩经验也适用于 LRU（用 HashMap+链表组合出 O(1) 缓存）、LFU（多层结构组合）、跳表（多级索引）。面试里遇到"用 X 实现 Y"的题，先想"Y 的语义需要什么操作，X 提供什么，缺的怎么补"，组合着补。
